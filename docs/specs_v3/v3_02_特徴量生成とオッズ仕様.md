# v3 特徴量生成とオッズ仕様

> **スコープ**: `build_features_v3.py`, `odds_v3_common.py`

---

## 1. 入力

- 主入力: `data/features_base.parquet`
- 必須列: `race_id`, `horse_no`, `target_label`, `race_date`, `field_size`

### 1.1 TE feature set

- binary の `feature_set=te` に使う `data/features_v3_te.parquet` は、`scripts_v3/build_features_v3_te.py` で生成する
- base は current `features_v3*.parquet`、TE source は `features_v2_te*.parquet` を前提とする
- build 時は `feature_registry_v3.py` の `get_binary_safe_te_feature_columns()` を source of truth とし、safe TE extra 列だけを追加する
- join key は `race_id`, `horse_id`, `horse_no` で固定する
- 出力は current `features_v3` の全列を保持しつつ、safe TE extra 列だけを追加した parquet と meta JSON にする
- `features_v3_te*.parquet` も current `features_v3*` と同じ stacker market 列を保持している必要がある

---

## 2. v3 で追加する主要列

### 2.1 ラベル・結果

| 列名 | 定義 | 型 |
|---|---|---|
| `y_win` | `target_label == 3`（1着） | int |
| `y_place` | `target_label >= 1`（3着以内） | int |
| `finish_pos` | DB `core.result` から結合 | float (`NaN` 許容) |

### 2.2 単勝 snapshot

| 列名 | 定義 |
|---|---|
| `odds_win_final` | final snapshot の単勝オッズ |
| `p_win_odds_final_raw` | `1 / odds_win_final` |
| `p_win_odds_final_norm` | レース内正規化した暗示確率 |
| `odds_win_t20` | `race_start - 20min` 時点で利用可能な最新単勝オッズ |
| `odds_win_t15` | `race_start - 15min` 時点で利用可能な最新単勝オッズ |
| `odds_win_t10` | `race_start - 10min` 時点で利用可能な最新単勝オッズ |
| `p_win_odds_t20_norm` | `1 / odds_win_t20` を race 内で再正規化した暗示確率 |
| `p_win_odds_t15_norm` | `1 / odds_win_t15` を race 内で再正規化した暗示確率 |
| `p_win_odds_t10_raw` | `1 / odds_win_t10` |
| `p_win_odds_t10_norm` | レース内正規化した暗示確率 |
| `d_logit_win_15_20` | `logit(p_win_odds_t15_norm) - logit(p_win_odds_t20_norm)` |
| `d_logit_win_10_15` | `logit(p_win_odds_t10_norm) - logit(p_win_odds_t15_norm)` |
| `d_logit_win_10_20` | `logit(p_win_odds_t10_norm) - logit(p_win_odds_t20_norm)` |
| `odds_win_t20_data_kbn`, `odds_win_t15_data_kbn`, `odds_t10_data_kbn` | 採用 snapshot の `data_kbn` |
| `odds_win_t20_announce_dt`, `odds_win_t15_announce_dt`, `odds_t10_announce_dt` | 採用 snapshot の発表時刻 |
| `odds_win_t20_asof_dt`, `odds_win_t15_asof_dt`, `odds_t10_asof_dt` | as-of 基準時点 |

### 2.3 複勝 snapshot

| 列名 | 定義 |
|---|---|
| `odds_place_t20_lower`, `odds_place_t20_upper` | t20 の複勝オッズ下限 / 上限 |
| `odds_place_t15_lower`, `odds_place_t15_upper` | t15 の複勝オッズ下限 / 上限 |
| `odds_place_t10_lower`, `odds_place_t10_upper` | t10 の複勝オッズ下限 / 上限 |
| `place_mid_prob_t20` | `1 / sqrt(low * high)` で作る t20 の複勝中心確率 |
| `place_mid_prob_t15` | `1 / sqrt(low * high)` で作る t15 の複勝中心確率 |
| `place_mid_prob_t10` | `1 / sqrt(low * high)` で作る t10 の複勝中心確率 |
| `place_width_log_ratio_t20` | `log(high / low)`。`low=min(lower, upper)`, `high=max(lower, upper)` |
| `place_width_log_ratio_t15` | `log(high / low)`。`low=min(lower, upper)`, `high=max(lower, upper)` |
| `place_width_log_ratio_t10` | `log(high / low)`。`low=min(lower, upper)`, `high=max(lower, upper)` |
| `d_place_mid_10_20` | `place_mid_prob_t10 - place_mid_prob_t20` |
| `d_place_width_10_20` | `place_width_log_ratio_t10 - place_width_log_ratio_t20` |
| `place_width_log_ratio` | default PL 用の正式列。`place_width_log_ratio_t10` と同義 |
| `odds_place_t20_data_kbn`, `odds_place_t15_data_kbn`, `odds_place_t10_data_kbn` | 採用 snapshot の `data_kbn` |
| `odds_place_t20_announce_dt`, `odds_place_t15_announce_dt`, `odds_place_t10_announce_dt` | 採用 snapshot の発表時刻 |
| `odds_place_t20_asof_dt`, `odds_place_t15_asof_dt`, `odds_place_t10_asof_dt` | as-of 基準時点 |

## 3. snapshot 方針

### 3.1 final（検証用）

- `data_kbn` 優先順位: `4 > 3 > 2 > 1`
- `announce <= race_start` の範囲で最新を採用
- 用途: 検証、比較、校正評価

### 3.2 fixed stacker snapshots

- as-of 基準は `t20`, `t15`, `t10` の 3 点
- いずれも `announce <= as_of` を満たす最新 snapshot を採用する
- default stacker で使い分ける入力は次で固定する
  - `stack_win`: `p_win_odds_t20_norm`, `p_win_odds_t15_norm`, `p_win_odds_t10_norm`, `d_logit_win_15_20`, `d_logit_win_10_15`, `d_logit_win_10_20`
  - `stack_place`: `place_mid_prob_t20/t15/t10`, `place_width_log_ratio_t20/t15/t10`, `d_place_mid_10_20`, `d_place_width_10_20`
- `stack_win` に複勝時系列は入れない
- `stack_place` に単勝時系列は入れない

### 3.3 欠損処理

- オッズが `NULL / 0 / 負` の場合は欠損扱い
- 単勝暗示確率は race 内で再正規化する
- 複勝の中心値 / 幅は `low=min(lower, upper)`, `high=max(lower, upper)` を使って計算する

---

## 4. リーク防止アサーション

- `assert_asof_no_future_reference()` により、win/place の `t20/t15/t10` すべてで `announce_dt <= asof_dt` を検証する
- `assert_t10_no_future_reference()` は後方互換の alias として残すが、default build / inference path では `assert_asof_no_future_reference()` を使う

---

## 5. feature table と feature contract の分離

- `features_v3` には final / `t20/t15/t10` の odds 列を保持してよい
- ただし、学習で実際に使う列は `feature_registry_v3.py` の contract で制御する
- binary default は `BINARY_BASE_FEATURES` のみを使い、odds 列は default contract に入れない
- stack default は raw odds / raw lower-upper ではなく、確率化した市場列と時間差分を優先する

---

## 6. メタデータ出力

`features_v3_meta.json` に少なくとも次を記録する。

- 入出力パス、行数、レース数、列一覧
- `operational_default = "t10_only"`
- `contains_final_odds_columns`
- `contains_t10_odds_columns`
- `contains_stacker_timeseries_columns`
- `finish_pos`, `p_win_odds_t20/t15/t10_norm`, `d_logit_win_*`, `place_mid_prob_t20/t15/t10`, `d_place_*`, `place_width_log_ratio` の coverage
- コードハッシュ（`build_features_v3.py` + `odds_v3_common.py`）

`features_v3_te_meta.json` は上記に加えて、少なくとも次を記録する。

- `te_source_input_path`
- `te_feature_columns`
- `te_join_keys`
- safe TE extra 列の coverage
