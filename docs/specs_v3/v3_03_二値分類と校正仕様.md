## 1. binary 学習アーキテクチャ

6つの薄いラッパースクリプトが `main(default_task=..., default_model=...)` を呼び出す。

```text
train_win_lgbm_v3.py   → main(task="win",   model="lgbm")
train_win_xgb_v3.py    → main(task="win",   model="xgb")
train_win_cat_v3.py    → main(task="win",   model="cat")
train_place_lgbm_v3.py → main(task="place", model="lgbm")
train_place_xgb_v3.py  → main(task="place", model="xgb")
train_place_cat_v3.py  → main(task="place", model="cat")
```

### 1.1 ラベル定義

| task | ラベル列 | 定義 | base rate (参考) |
|---|---|---|---|
| `win` | `y_win` | 1着（`target_label == 3`） | ~6-7% |
| `place` | `y_place` | 3着以内（`target_label >= 1`） | ~20-21% |

## 2. feature registry 契約

binary の特徴量選択は `scripts_v3/feature_registry_v3.py` の whitelist を使う。

### 2.1 default contract

- default operational profile は `t10_only`
- default feature set は `BINARY_BASE_FEATURES + BINARY_T10_ODDS_FEATURES`
- `jockey_key`, `trainer_key` は `--include-entity-id-features` 指定時だけ追加する
- final odds 系は `--operational-mode includes_final` 指定時だけ追加する
- `finish_pos`, `target_label`, `y_win`, `y_place` など post-race / label 列は contract で禁止する

### 2.2 manifest 出力

`models/{task}_{model}_bundle_meta_v3.json` と `models/{task}_{model}_feature_manifest_v3.json` に少なくとも以下を保存する。

- `operational_mode`
- `include_entity_id_features`
- `feature_columns`
- `categorical_features`
- `forbidden_feature_check_passed`
- `feature_manifest_version`

## 3. Cross-Validation

- 方式: Rolling 年次 CV（`build_rolling_year_folds()`）
- default: `train_window_years=5`, `holdout_year=2025`
- 整合性チェック: `assert_fold_integrity()` で train / valid 間の年重複・`race_id` 重複を検出

### 3.1 fold 生成イメージ

```text
holdout_year=2025, train_window_years=5 の場合:
  fold 0: train=[2016,2017,2018,2019,2020] → valid=2021
  fold 1: train=[2017,2018,2019,2020,2021] → valid=2022
  fold 2: train=[2018,2019,2020,2021,2022] → valid=2023
  fold 3: train=[2019,2020,2021,2022,2023] → valid=2024
year >= holdout_year は CV・最終学習から除外
```

## 5. 最終モデル

CV 完了後、以下の2つの最終モデルを学習する。

1. `recent_window`: 直近 `train_window_years + 1` 年分のデータで学習（運用推論用）
2. `all_years`: holdout 除外後の全年データで学習（比較・分析用途）

最終モデルのブースト回数は CV fold の `best_iteration` の中央値を使用する。

## 8. Benter R*

### 8.1 対象

- 単勝タスク（`task="win"`）のみ
- 各 fold の valid 上で計算する

### 8.2 手順

1. 予測確率 `p` を clip して logit 変換する  
   `score = logit(clip(p, ε, 1-ε))`, `ε = 1e-6`
2. レース内 softmax（温度 β）を計算する  
   `c_i(β) = exp(β·s_i) / Σ_j exp(β·s_j)`
3. モデル NLL を計算する  
   `NLL_model = -Σ_race log(c_winner)`
4. ヌル NLL を計算する  
   `NLL_null = Σ_race log(field_size)`
5. `R* = 1 - NLL_model / NLL_null`

### 8.3 β の最適化

- 対象データ: fold 内の train 予測のみ（in-sample）
- 探索方法:
  1. log スケール等間隔グリッド（81点, `[0.01, 100.0]`）
  2. 最良グリッド点の前後区間で黄金分割探索（48反復）
- 報告:
  - `β̂` での `R*`
  - `β=1` での `R*`

## 9. オッズ確率校正

### 9.1 概要

オッズ由来の暗示確率（`p_win_odds_t10_norm`, `p_win_odds_final_norm`）を `y_win` に対して校正する。

### 9.2 校正手法

| 手法 | 実装 |
|---|---|
| `logreg` | `LogisticRegression(C=1e6)` で `logit(p)` → `y_win` を学習 |
| `isotonic` | `IsotonicRegression(out_of_bounds="clip")` で `p` → `y_win` を学習 |

### 9.3 CV 方式

- binary と同一の Rolling 年次 CV（default `train_window_years=5`, `holdout_year=2025`）

### 9.4 出力

- OOF: `data/oof/odds_win_calibration_oof.parquet`
  - 各 `(score_col, method)` の組み合わせについて校正後確率列を追加する
- メトリクス: `data/oof/odds_win_calibration_cv_metrics.json`
- モデル: `models/odds_win_calibrators_v3.pkl`
