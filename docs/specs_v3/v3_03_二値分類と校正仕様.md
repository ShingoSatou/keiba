# v3 二値分類・stacker・校正仕様

> **スコープ**: `feature_registry_v3.py`, `train_binary_model_v3.py`, `train_stacker_v3_common.py`, `metrics_benter_v3_common.py`, `train_odds_calibrator_v3.py`

---

## 1. base binary 学習

6 つの薄いラッパースクリプトが `main(default_task=..., default_model=...)` を呼び出す。

```text
train_win_lgbm_v3.py   → main(task="win",   model="lgbm")
train_win_xgb_v3.py    → main(task="win",   model="xgb")
train_win_cat_v3.py    → main(task="win",   model="cat")
train_place_lgbm_v3.py → main(task="place", model="lgbm")
train_place_xgb_v3.py  → main(task="place", model="xgb")
train_place_cat_v3.py  → main(task="place", model="cat")
```

### 1.1 ラベル定義

| task | ラベル列 | 定義 |
|---|---|---|
| `win` | `y_win` | 1着（`target_label == 3`） |
| `place` | `y_place` | 3着以内（`target_label >= 1`） |

### 1.2 base 出力契約

後段で使う base 予測列名は次で固定する。

- `p_win_lgbm`
- `p_win_xgb`
- `p_win_cat`
- `p_place_lgbm`
- `p_place_xgb`
- `p_place_cat`

OOF / holdout / inference で同一列名を使う。

## 2. binary feature registry 契約

- default operational profile は `t10_only`
- default feature set は `BINARY_BASE_FEATURES + BINARY_T10_ODDS_FEATURES`
- `jockey_key`, `trainer_key` は `--include-entity-id-features` 指定時だけ追加する
- final odds 系は `--operational-mode includes_final` 指定時だけ追加する
- `finish_pos`, `target_label`, `y_win`, `y_place` など post-race / label 列は contract で禁止する

---

## 3. binary CV

- 方式: fixed-length sliding yearly CV
- default: `train_window_years=4`, `holdout_year=2025`, `cv_window_policy=fixed_sliding`
- 4 年未満しか履歴がない初期年は fold を作らない
- 整合性チェック: `assert_fold_integrity()`

### 3.1 fold 生成イメージ

```text
holdout_year=2025, train_window_years=4 の場合:
  fold 0: train=[2016,2017,2018,2019] → valid=2020
  fold 1: train=[2017,2018,2019,2020] → valid=2021
  fold 2: train=[2018,2019,2020,2021] → valid=2022
  fold 3: train=[2019,2020,2021,2022] → valid=2023
  fold 4: train=[2020,2021,2022,2023] → valid=2024
```

## 4. strict temporal stacker

### 4.1 役割

stacker は grouped reference-only meta の代わりに、horse-level の strict temporal 予測を作る層である。

- `stack_win` → `p_win_stack`
- `stack_place` → `p_place_stack`

PL の default main path はこの 2 列を主入力に使う。

### 4.2 stacker 入力契約

#### `stack_win`

- `p_win_lgbm`
- `p_win_xgb`
- `p_win_cat`
- `odds_win_t20`
- `odds_win_t15`
- `odds_win_t10`
- `track_code`
- `surface`
- `distance_m`
- `going`
- `weather`
- `field_size`
- `grade_code`
- `race_type_code`
- `weight_type_code`
- `condition_code_min_age`

#### `stack_place`

- `p_place_lgbm`
- `p_place_xgb`
- `p_place_cat`
- `odds_place_t20_lower`
- `odds_place_t20_upper`
- `place_width_log_ratio_t20`
- `odds_place_t15_lower`
- `odds_place_t15_upper`
- `place_width_log_ratio_t15`
- `odds_place_t10_lower`
- `odds_place_t10_upper`
- `place_width_log_ratio_t10`
- `track_code`
- `surface`
- `distance_m`
- `going`
- `weather`
- `field_size`
- `grade_code`
- `race_type_code`
- `weight_type_code`
- `condition_code_min_age`

### 4.3 stacker 学習器

- current implementation は `LightGBM`
- artifact は `models/{task}_stack_v3.txt`, `models/{task}_stack_all_years_v3.txt`, `models/{task}_stack_bundle_meta_v3.json`

### 4.4 stacker CV

- 方式: strict temporal `capped_expanding`
- 窓: `min_train_years=2`, `max_train_years=4`
- 挙動:
  - 2 年学習 → 次年 valid
  - 3 年学習 → 次年 valid
  - 4 年学習 → 次年 valid
  - 以後は直近 4 年 rolling

例:

```text
train=[2020,2021]             → valid=2022
train=[2020,2021,2022]        → valid=2023
train=[2020,2021,2022,2023]   → valid=2024
train=[2021,2022,2023,2024]   → valid=2025
```

### 4.5 リーク防止条件

- downstream に渡す stacker 予測は OOF / holdout / inference のみ
- train 同年の fitted 値を PL 学習に混ぜない
- stacker bundle meta には `base_oof_valid_years` と `stacker_oof_valid_years` を保存する

## 6. 最終モデル

binary / stacker とも CV 完了後に次の 2 つの最終モデルを学習する。

1. `recent_window`: holdout 年直前の window のみで学習
2. `all_years`: holdout 除外後の全年データで学習（analysis-only）

stacker の recent window は capped expanding の `max_train_years=4` に合わせる。

## 9. Benter R*

- 対象: 単勝 task（`task="win"`）のみ
- 各 fold の valid 上で計算する
- score は `logit(clip(p, ε, 1-ε))`, `ε = 1e-6`

---

## 10. オッズ確率校正

### 10.1 概要

オッズ由来の暗示確率（`p_win_odds_t10_norm`, `p_win_odds_final_norm`）を `y_win` に対して校正する。

### 10.2 校正手法

| 手法 | 実装 |
|---|---|
| `logreg` | `LogisticRegression(C=1e6)` |
| `isotonic` | `IsotonicRegression(out_of_bounds="clip")` |

### 10.3 位置づけ

- odds 校正は comparison / analysis 用の optional path
- default stacker / PL main path の必須前段ではない
