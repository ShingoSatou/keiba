# v3 実装仕様（単勝/複勝/odds/PL）

## 1. 目的

v2のデータ設計・リーク防止・Rolling年次CV・OOF保存作法を踏襲しつつ、以下を追加する。

1. 単勝2値分類（`y_win`）: `lgbm/xgb/cat`
2. 複勝2値分類（`y_place`）: `lgbm/xgb/cat`
3. オッズ確率（final / t10）と必要に応じた校正
4. PLレイヤ（uなし、`s=w^T x`）で `p_top3` を推定

## 2. 入力データ

- 主入力: `data/features_v2.parquet`
- v3特徴量出力: `data/features_v3.parquet`
  - 追加列:
    - `y_win = (target_label == 3)`
    - `y_place = (target_label >= 1)`
    - `finish_pos`（DB結合）
    - `odds_win_final`, `p_win_odds_final_raw`, `p_win_odds_final_norm`
    - `odds_win_t10`, `p_win_odds_t10_raw`, `p_win_odds_t10_norm`

## 3. CV/OOF方針

- split関数は v2 `build_rolling_year_folds()` を利用
- holdout除外ルール: `year >= holdout_year` はCV・最終学習から除外
- OOFは `data/oof/*.parquet` + fold metrics `data/oof/*_cv_metrics.json`
- 後段（PL入力）には OOF 予測のみを使用

## 4. Benter R*（単勝のみ）

- 対象: `p_win_*`（単勝分類）
- 手順:
  1. `p` を clip して `score=logit(p)` を作成
  2. レース内 softmax（温度 `beta`）で `c_i(beta)` へ変換
  3. `NLL_model = -Σ log c_winner`
  4. `NLL_null = Σ log(field_size)`
  5. `R* = 1 - NLL_model / NLL_null`
- `beta` は fold内 train のみで最適化（grid + golden-section, SciPy不使用）

## 5. odds snapshot方針

- final（検証用）:
  - `data_kbn=4` 優先、なければ `3 -> 2 -> 1`
  - `announce <= race_start` の範囲で最新を採用
- t10（運用想定）:
  - `as_of = race_start - 10min`
  - `announce <= as_of` の範囲で最新を採用
- 欠損時は race内で再正規化（全欠損ならNaN）

## 6. PL（uなし）

- スコア: `s_{r,i} = w^T x_{r,i}`
- 学習: 観測着順に対するPL負の対数尤度
- 実装: PyTorch（optional dependency）
- 推論:
  - 必須: `p_top3`（Monte Carlo）
  - 任意: `p_wide(i,j)`（Monte Carlo）

## 7. 運用パス

- `predict_race_v3.py` は運用想定として t10 パスを採用
- PLモデルが final odds 系列（`*_final_*`）を要求する場合、推論時にエラーで停止
