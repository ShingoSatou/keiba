# v3 仕様書インデックス

> **ステータス**: 2026-03-06 に monolithic 仕様書を分割
> **対象**: `scripts_v3/` 配下の全スクリプト + `docs/ops_v3/` 運用ドキュメント

このファイルは v3 仕様書の入口であり、作業内容ごとにどの仕様書を確認すべきかを示す台帳として使う。

---

## 1. 読み方

1. まず本ファイルで、対象作業に対応する仕様書を特定する。
2. 次に、該当する詳細仕様書を読む。
3. 最後に、`docs/ops_v3/` の運用文書で current default / 実行手順 / 実測結果を確認する。

## 4. Source Of Truth

- アーキテクチャ、データ契約、責務分割は `docs/specs_v3/` の分割仕様書を正とする。
- current default の運用プロファイル、CLI 実行入口、実行時の注意点は `docs/ops_v3/Assumptions.md` と `docs/ops_v3/スクリプトリファレンス.md` を正とする。
- 実測値、再学習結果、評価結果は `docs/ops_v3/v3_run_report_*.md` を正とする。
- 仕様と実装に差異が見つかった場合は、コードだけでなく該当する仕様書と ops docs をセットで更新する。
