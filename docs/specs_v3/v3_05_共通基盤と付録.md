## 1. 共通ユーティリティ

### 1.1 パス管理

- `resolve_path(path_str)`: 相対パスを `PROJECT_ROOT` 基準で解決する
- `save_json(path, payload)`: JSON 書き出し（`indent=2`, `ensure_ascii=False`）
- `hash_files(paths)`: SHA-256 ダイジェスト

### 1.2 Rolling 年次 CV

`build_fixed_window_year_folds(years, *, window_years, holdout_year) -> list[FoldSpec]`

`build_rolling_year_folds(years, *, train_window_years, holdout_year)` は互換 alias として残す。

- `holdout_year` 以降の年を除外する
- 各 fold の train は valid year 直前の固定長 window のみを使う
- train 窓が `window_years` に満たない年は fold を生成せずスキップする
- `FoldSpec(fold_id, train_years, valid_year)` のリストを返す

stacker 用には `build_capped_expanding_year_folds(years, *, min_window_years, max_window_years, holdout_year)` を使う。

- 初期は 2 年学習から開始する
- 3 年, 4 年へ段階的に拡張する
- 4 年到達後は直近 4 年 rolling を維持する

### 1.3 整合性アサーション

- `assert_fold_integrity(train, valid, valid_year)`: 年重複・`race_id` 重複の検出
- `assert_sorted(df)`: `race_id asc / horse_no asc` のソート検証

### 1.4 Bankroll / Kelly

- `kelly_fraction()`, `fractional_kelly_fraction()`
- `allocate_race_bets()`: レース内候補の Kelly ベット配分
- `compute_max_drawdown()`: エクイティカーブから最大 DD を計算

## 3. 既知の制限事項

1. registry を変更した場合は再学習が必要
2. PL は馬ID固定効果 `u` を持たない
3. `torch` 未導入環境では NumPy フォールバックとなり、学習品質が変わりうる
4. 2025 holdout は既に評価に使っているため、one-shot 性は弱まっている
5. current strict-temporal stacker coverage では、`holdout_year=2025` / `PL train_window_years=3` の OOF fold は 0 件になる
