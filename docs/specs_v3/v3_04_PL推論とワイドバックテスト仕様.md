# v3 PL推論とワイドバックテスト仕様

> **スコープ**: `train_pl_v3.py`, `pl_v3_common.py`, `predict_race_v3.py`, `backtest_wide_v3.py`

---

## 1. PL の位置づけ

PL は base 予測の統合器ではなく、**stacker 出力を入力として順位分布を学習するレイヤ**として扱う。

- 前段で horse-level の `p_win_stack`, `p_place_stack` を strict temporal に作る
- 後段の PL はこれを順位分布化し、`pl_score`, `p_top3`, `p_wide` を生成する
- grouped reference-only meta は comparison profile に残すが、default main path では使わない

---

## 2. Plackett-Luce ランキング

### 2.1 モデル定義

- スコア関数: `s_{r,i} = w^T x_{r,i}`（線形、馬 ID 固定効果 `u` なし）
- 損失関数: Plackett-Luce 負の対数尤度

```text
NLL = -Σ_race Σ_{k=1}^{n-1} [s_{π(k)} - log(Σ_{j=k}^{n} exp(s_{π(j)}))]
```

### 2.2 最適化

| 項目 | 値 |
|---|---|
| バックエンド | PyTorch（optional）、なければ NumPy fallback |
| epoch | 300 |
| learning rate | 0.05 |
| L2 | 1e-5 |
| seed | 42 |

## 3. PL 入力特徴量

### 3.1 default profile

default `pl_feature_profile` は `stack_default` とする。

PL 入力は「4 列」ではなく「4 グループ」で管理する。

#### group 1: 単勝強さ

- `z_win_stack = logit(clip(p_win_stack, eps, 1-eps))`

#### group 2: 複勝強さ

- `z_place_stack = logit(clip(p_place_stack, eps, 1-eps))`

#### group 3: 複勝幅

- `place_width_log_ratio = log(upper / lower)`
- default は `place_width_log_ratio_t10` を正式列 `place_width_log_ratio` として使う

#### group 4: interaction block

race context は独立 block を作らず、この interaction block に含める。current contract は少なくとも次を含む。

- context raw:
  - `track_code`
  - `surface`
  - `distance_m`
  - `going`
  - `weather`
  - `field_size`
  - `grade_code`
  - `race_type_code`
  - `weight_type_code`
  - `condition_code_min_age`
- interaction:
  - `z_win_stack_x_z_place_stack`
  - `z_win_stack_x_place_width_log_ratio`
  - `z_place_stack_x_place_width_log_ratio`
  - `z_win_stack_x_field_size`
  - `z_place_stack_x_field_size`
  - `z_win_stack_x_distance_m`
  - `z_place_stack_x_distance_m`
  - `z_win_stack_race_centered`
  - `z_place_stack_race_centered`
  - `place_width_log_ratio_race_centered`
  - `z_win_stack_rank_pct`
  - `z_place_stack_rank_pct`
  - `place_width_log_ratio_rank_pct`

### 3.2 comparison profile

- `meta_default`: grouped reference-only meta を入力に使う比較用 path

unsuffixed artifact / output path は `stack_default` に対応し、legacy profile は suffix 付き path を使う。

### 3.3 OOF 原則

- PL 学習には OOF 予測のみを使う
- `stack_default` では `p_win_stack`, `p_place_stack` の strict temporal OOF が揃った行だけを学習対象にする
- 同年 fitted 値を downstream に混ぜない

---

## 4. 前処理

- 確率列はそのまま使わず、必ず次の順で前処理する
  1. `p := clip(p, eps, 1 - eps)`
  2. `z := log(p / (1 - p))`
- 欠損値: train fold の中央値で補完
- 標準化: train fold の平均 / 標準偏差を使う
- 標準偏差が 0 の列はスケーリングをスキップする

---

## 5. PL CV 方式

- 方式: fixed-length sliding yearly CV
- default: `train_window_years=3`, `holdout_year=2025`, `cv_window_policy=fixed_sliding`
- outer fold の train は valid year 直前の 3 年のみを使う

### 5.1 前提条件

PL fixed3 を成立させるには、valid year に対して直前 3 年分の stacker OOF が必要である。実装は少なくとも次を確認し、`v3_pipeline_year_coverage.json` に保存する。

- `base_oof_years`
- `stacker_oof_years`
- `pl_eligible_years`
- `pl_oof_valid_years`
- `pl_holdout_train_years`
- `holdout_year`

### 5.2 OOF fold が 0 件のとき

current repo の年範囲では `base OOF=2020-2024`, `stacker OOF=2022-2024` となるため、`holdout_year=2025` / `train_window_years=3` では PL OOF fold が 0 件になる。

この場合の挙動は次で固定する。

- 学習を失敗にしない
- `pl_v3_oof.parquet` は空 schema で保存する
- `pl_v3_cv_metrics.json` と `v3_pipeline_year_coverage.json` に理由を記録する
- holdout scoring と final artifact 生成は継続する

## 7. 1レース推論

### 7.1 default path

```text
入力（1レース特徴量）
  → base model 推論（win/place × lgbm/xgb/cat）
  → stacker 推論（p_win_stack, p_place_stack）
  → PL feature materialization（z/logit + interaction block）
  → PL scoring（pl_score）
  → Monte Carlo（p_top3, p_wide）
  → 出力
```

### 7.2 制約

- 入力は 1 レースのみ
- as-of 契約は `assert_asof_no_future_reference()` で検証する
- final odds を含む PL artifact は運用推論で拒否する
- `stack_default` では stacker bundle meta から `p_win_stack`, `p_place_stack` を生成できる
- `meta_default` は comparison artifact として残す

### 7.3 主な出力列

```text
race_id, horse_id, horse_no, race_date,
p_win_lgbm, p_win_xgb, p_win_cat,
p_place_lgbm, p_place_xgb, p_place_cat,
p_win_stack, p_place_stack,
z_win_stack, z_place_stack, place_width_log_ratio,
pl_score, p_top3
```

## 8. Monte Carlo 推定

### 8.1 p_top3

- `pl_score` に Gumbel(0,1) を加算して順位サンプルを作る
- `top_k=3`, `mc_samples=10_000`

### 8.2 p_wide

- 同じサンプルから 2 頭がともに top-3 に入る頻度を推定する
- 出力は `(race_id, horse_no_1, horse_no_2, p_wide)`

### 8.3 seed

- `race_id + global_seed` から決定論的 RNG を作る

## 9. ワイドバックテスト

### 9.1 入力モード

| モード | 判定条件 | 内容 |
|---|---|---|
| pair-level | `p_wide` 列が存在 | `p_wide` を直接使用 |
| horse-level | `pl_score` 列が存在 | `pl_score -> MC -> p_wide` を再計算 |

### 9.2 標準評価経路

- v3 標準評価は `backtest_wide_v3.py`
- `pl_score -> MC -> p_wide` もしくは `train_pl_v3.py --emit-wide-oof` の `p_wide` を使う
