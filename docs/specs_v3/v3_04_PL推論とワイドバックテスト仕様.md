> **スコープ**: `train_pl_v3.py`, `pl_v3_common.py`, `predict_race_v3.py`, `backtest_wide_v3.py`

---

## 1. Plackett-Luce ランキング

### 1.1 モデル定義

- スコア関数: `s_{r,i} = w^T x_{r,i}`（線形、馬ID固定効果 `u` なし）
- 損失関数: Plackett-Luce 負の対数尤度

```text
NLL = -Σ_race Σ_{k=1}^{n-1} [s_{π(k)} - log(Σ_{j=k}^{n} exp(s_{π(j)}))]
```

ここで `π` は観測着順を表す。

### 1.2 最適化

| 項目 | 値 |
|---|---|
| バックエンド | PyTorch（optional）、なければ NumPy gradient descent フォールバック |
| エポック数 | 300 |
| 学習率 | 0.05 |
| L2正則化 | 1e-5 |
| seed | 42 |

#### NumPy フォールバック

- 勾配を手動計算する
- 学習率を Adam ライクに動的調整する（`1e-3 ~ 5e-5` の段階的減衰）

---

## 2. PL 入力特徴量

PL 学習の入力は以下を統合する。

1. 前段 OOF 予測: `p_win_{lgbm,xgb,cat}`, `p_place_{lgbm,xgb,cat}`
2. オッズ校正結果（任意）: `p_win_odds_*_cal_*`
3. t10 odds: `PL_T10_ODDS_FEATURES`
4. small context: `PL_CONTEXT_FEATURES_SMALL`
5. final odds 列: `--include-final-odds-features` 指定時だけ追加

### 2.1 default contract

- default operational profile は `t10_only`
- default feature list は `required OOF + t10 odds + small context`
- `features_v3` の numeric 列を総なめして自動収集することはしない
- `jockey_key`, `trainer_key`, `finish_pos`, label 系は PL feature contract に含めない

> [!IMPORTANT]
> PL 学習入力には **OOF 予測のみ** を使用する。OOF が欠ける行は PL 学習対象から除外される。

---

## 3. 前処理

- 欠損値: 各特徴量の中央値で補完
- 標準化: 平均0・標準偏差1（train fold の統計量を使用）
- 標準偏差が0の特徴量はスケーリングをスキップする

---

## 4. CV 方式

- fixed-length sliding yearly CV（binary と同一構造）
- current default: `train_window_years=4`, `cv_window_policy=fixed_sliding`
- holdout 除外ルールも binary と同一
- 4年未満しか履歴がない初期年は fold を作らずスキップする

---

## 5. Monte Carlo 推定

### 5.1 p_top3

PL 分布からの独立サンプリングで、各馬が top-3 に入る確率を Monte Carlo 推定する。

```python
mc_samples = 10_000
top_k = 3
```

1. `pl_score` から Gumbel(0,1) を加算して順位サンプルを生成する
2. 各サンプルで top-k に入る回数をカウントする
3. 出現頻度を確率として推定する

### 5.2 p_wide

2頭の組み合わせ `(i, j)` がともに top-3 に入る確率を推定する。

- 上記 MC サンプルの副産物として計算する
- 出力は `(race_id, horse_no_1, horse_no_2, p_wide)` のペアレベル

### 5.3 乱数シード

レース再現性のため `race_id + global_seed` から決定論的 RNG を生成する。

```python
rng = np.random.default_rng(
    int(hashlib.sha256(f"{seed}_{race_id}".encode()).hexdigest()[:8], 16) % (2**32)
)
```

## 7. 1レース推論

### 7.1 運用パス

```text
入力（1レース特徴量）
  → base model 推論（win/place × lgbm/xgb/cat → 6列追加）
  → odds 校正（任意）
  → PL scoring（w^T x → pl_score）
  → Monte Carlo 推定 → p_top3, （任意）p_wide
  → 出力
```

### 7.2 制約事項

- 入力は1レースのみ（`race_id` が1種類であること）
- t10 パス強制: PL モデルの特徴量に `_final_` を含む列があればエラーで停止
- base model meta から特徴量列・モデルパスを読み取る

### 7.3 出力列

```text
race_id, horse_id, horse_no, race_date,
odds_win_t10, p_win_odds_t10_raw, p_win_odds_t10_norm,
p_win_lgbm, p_win_xgb, p_win_cat,
p_place_lgbm, p_place_xgb, p_place_cat,
pl_score, p_top3
```

- `--emit-wide` 指定時はペアレベル `(race_id, horse_no_1, horse_no_2, p_wide)` を追加出力する

## 8. ワイドバックテスト

### 8.1 概要

PL 由来の `p_wide` を基に、ワイド馬券の購入シミュレーションと ROI 評価を行う。

### 8.2 入力モード

自動検出により2モードを切り替える。

| モード | 判定条件 | 内容 |
|---|---|---|
| **pair-level** | `p_wide` 列が存在 | `(race_id, horse_no_1, horse_no_2, p_wide)` を直接使用 |
| **horse-level** | `pl_score` 列が存在 | `pl_score` → MC 推定 → `p_wide` を算出 |

### 8.3 データソース

| テーブル | 用途 |
|---|---|
| `core.o3_wide` | ワイドオッズ取得 |
| `core.payout` (`bet_type=5`) | ワイド払戻金取得 |
| `core.race` | レース日・開催時刻 |
| `core.runner` | horse_name 取得 |

### 8.4 Bankroll 管理

```python
@dataclass
class BankrollConfig:
    bankroll_init_yen: int = 1_000_000
    kelly_fraction_scale: float = 0.25
    max_bets_per_race: int = 5
    race_cap_fraction: float = 0.05
    daily_cap_fraction: float = 0.2
    bet_unit_yen: int = 100
    min_bet_yen: int = 100
    max_bet_yen: int | None = None
```

#### Kelly fraction 計算

```python
def kelly_fraction(probability, odds):
    edge = probability * odds - 1.0
    if edge <= 0.0:
        return 0.0
    return edge / (odds - 1.0)

fractional_kelly = kelly_fraction * kelly_fraction_scale
```

#### レース内配分

1. 候補ペアを `p_wide * odds - 1 > 0` でフィルタする
2. `p_wide` 閾値でさらにフィルタする
3. Kelly fraction × 資金からベット額を計算する
4. レース上限、日次上限で制限する
5. 100円単位に切り捨てる

### 8.5 シミュレーション

- レースを日付順に処理する
- 各レースで「ベット → 払戻 → 資金更新」を繰り返す
- 月次集計（投入額、回収額、ROI、残高推移）を出力する

### 8.6 出力

| `backtest_wide_v3_*.json` | サマリ、月次明細、全ベット記録 |
| `backtest_wide_v3_*_meta.json` | 設定、品質指標、ドローダウン |
