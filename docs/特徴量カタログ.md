# 特徴量カタログ（単勝・T-5）v1.2

対象：学習/検証期間 2016年〜現在（0B41はバックフィル、Realtime-onlyは運用開始日以降）

## 0. 共通ルール（全特徴量に適用）

### 0.0 Backfillable / Realtime-only（重要）

* **Backfillable（再現学習に使える）**：2016〜現在を再現可能に学習・検証できる特徴量
  * JVOpen（蓄積系）が中心
  * **0B41（時系列オッズ）は JVRTOpen の racekey 指定でバックフィル**する（参照: `docs/ダウンロードリスト.md`, `docs/jvlink_handover.md`）
* **Realtime-only（運用開始以降のみ）**：運用当日の監視/スナップ用途で取得するが、長期バックフィルは前提にしない特徴量
  * 例：0B11（WH）, 0B14（WE/AV/JC/TC/CC）, 0B13/0B17（速報DM/TM）

> 本カタログは「求め方」を固定しますが、**データが存在しない期間は欠損（または未使用）**になります。
> 0B41はバックフィルで埋める前提とし、0B11/0B14/0B13/0B17 依存の特徴量は現時点では運用開始日以降のみ（必要なら別途方針を定める）。

### 0.1 レースキー・馬キー

* `race_key`：`YYYYMMDDJJKKHHRR`（または内部で等価なキー）
* `horse_no`：馬番（01〜18など）

### 0.2 as-of（時点固定）ルール

* 推論・学習ともに **基準時刻 = 発走時刻 T の 5分前（T-5）**
* あるデータが複数発表される場合：
  **「T-5 以前で最も新しい発表」**を採用（採用したキー/時刻も保存）

### 0.3 O1（オッズ）スナップショットの厳密キー

* O1のデータ区分：`1:中間 / 2:前日売最終 / 3:最終 / 4:確定 / 5:確定(月曜)`
* **発表月日時分（MMDDhhmm）**は

  * **中間オッズのみ設定**
  * **時系列オッズを使用する場合のみキーとして設定**

> よって、T-5スナップは原則 **O1の data_kbn=1（中間）**から、
> `announce_mmddhhmi <= (T-5)` を満たす最大の `announce_mmddhhmi` を選ぶ。

### 0.4 O1オッズ値の特殊値→欠損/状態フラグ

（単勝オッズの項目仕様より）

* `"9999"`：999.9倍以上
* `"0000"`：無投票
* `"----"`：発売前取消
* `"****"`：発売後取消
* `" "`：登録なし

→ 取込/特徴量化では

* `core.o1_win.win_odds_x10`:
  * `*` を含む値、数字が無い値は `NULL`（欠損）として保存
  * `0000` は `0` として保存
* `odds_numeric`（`odds_win_t5`）:
  * `win_odds_x10 IS NULL OR win_odds_x10 <= 0` は `NULL`（欠損）
* `odds_state_flag`：上記状態を one-hot（or カテゴリ）で保持

### 0.5 O1の票数合計（レース全体）

O1には **単勝票数合計（返還分票数を含む）**等が存在
→ 票数の“時系列”は（手持ちデータ確認に基づき）O1時系列で利用可、ただし **馬別票数が取れない場合は合計のみ**を使う。

## 1. データソース（レコード）と採用タイミングの固定

### 1.1 時系列オッズ

* `dataspec=0B41` は **時系列オッズ（単複枠）**で、レコード種別は **O1**

### 1.2 DM/TM（データマイニング予想）

* DM/TMのデータ区分：`1:前日 / 2:当日 / 3:直前(馬体重発表後) / 7:成績(月曜) / 0:削除`
  ※`7`は蓄積系のみ、`7`提供後も速報系には `3` が提供
* TMの馬別指標：**予測スコア 000.0〜100.0**（小数1桁）
* DMの馬別指標：**予想走破タイム**と **予想誤差（＋/−）**が入る

**採用ルール（運用T-5）**
`kbn=3 → 2 → 1` の優先で、同一kbn内は「データ作成時分 最大」を採用。
（学習は原則 MING を正：後述）

### 1.3 MING（蓄積マイニング：学習の正）

* MINGは **直前予想（馬体重発表後）時点**のDM/TMを **月曜14:00頃**に提供
  → 学習・検証では **MING（=確定スナップ）を優先採用**し、速報の上書き問題を回避。

### 1.4 CK（出走別着度数）

* CK（着度数情報）は **出走馬名表発表時点**の各種着度数として提供
* CKには、距離帯（芝1600以下/1601-2200/2201以上、ダも同様）や脚質傾向（逃げ/先行/差し/追込回数）等が定義されている

## 2E. スピードレーティング（F/M 共通）

仕様として「自作SR」を固定します（JRA-VAN側に別レーティングがある場合は `F_sr_jvan_*` として同時採用してOK）。

### E0. 自作SR（推奨定義：パータイム＋馬場差補正）

**目的**：異なるレース条件の走破タイムを共通尺度へ。

#### E0-1. 必要な中間量

* `raw_time_sec`：走破タイム（秒）
* `distance_m`
* `class_bucket`（条件/格）
* `track_id + surface + turn`
* `going`（良/稍/重/不）
* `carried_weight`

#### E0-2. パータイムとトラック・バリアント

* `par_time_sec(track_id, surface, turn, distance_m, class_bucket)`

  * 過去データから「中央値（or 回帰）」で作る
* `variant_sec(race_day, track_id, surface)`

  * 同日の複数レースから「平均との差」で推定

#### E0-3. SR計算式（固定）

* `adj_time = raw_time_sec - variant_sec - w_coef*(carried_weight - std_weight)`
* `SR = SR_BASE + SR_SCALE * (par_time_sec - adj_time)`

  * 例：`SR_BASE=50`, `SR_SCALE=10`（後で調整）

### E1. SR特徴量（導入確定）

* `F_sr_last`：前走SR
* `F_sr_best_12m`：過去12ヶ月最高SR
* `F_sr_mean_5`：直近5走平均SR
* `F_sr_trend_3`：直近3走のSR傾き（回帰直線 slope）
* `F_sr_course_dist`：同コース×距離のSR平均（なければ全体平均）

## 2F. CK（出走別着度数）ミニ特徴量（確定45列）＋派生率

ここは「以前決めたCKミニセット」を **そのまま仕様として固定**します。
（CKは出走馬名表発表時点の着度数で提供）

### F0. 共通の作り方（CKブロック→countsへ）

CKにある各「着回数」ブロックは

* `count_1..count_5, count_out`（6着以下）
  で持つ前提。距離帯や脚質も仕様にある。

**基本集約（すべてのブロックで共通）**

* `starts = c1+c2+c3+c4+c5+cout`
* `wins = c1`
* `top3 = c1+c2+c3`
* `top5 = c1+...+c5`
* `out = cout`

**率（派生。学習入力に入れる）**

* `win_rate = (wins + α) / (starts + α+β)`
* `top3_rate = (top3 + α3) / (starts + α3+β3)`
* `top5_rate = (top5 + α5) / (starts + α5+β5)`

> 平滑化は固定：例として `α=1, β=9`（ベータ事前分布）
> ※母数が小さい時の暴れ対策（リークではない）

### F1. CK 45列（“カウント”として保存する列）

（以前の45列＝「総合/中央」「回り」「馬場」「距離」「脚質」「登録レース数」「賞金」）

#### F1-1. 距離帯（芝/ダ：〜1600、1601-2200、2201〜）

* `CK_turf_16down_*`
* `CK_turf_22down_*`
* `CK_turf_22up_*`
* `CK_dirt_16down_*`
* `CK_dirt_22down_*`
* `CK_dirt_22up_*`
  （CKの距離別着回数ブロック仕様）

#### F1-2. 脚質傾向（逃げ/先行/差し/追込 回数）

* `CK_style_nige_cnt`
* `CK_style_senko_cnt`
* `CK_style_sashi_cnt`
* `CK_style_oikomi_cnt`
  （脚質傾向の仕様）

#### F1-3. 馬場状態別（芝/ダ：良/稍/重/不）

* `CK_turf_good_*`, `CK_turf_soft_*`, `CK_turf_heavy_*`, `CK_turf_bad_*`
* `CK_dirt_good_*`, `CK_dirt_soft_*`, `CK_dirt_heavy_*`, `CK_dirt_bad_*`
  （馬場状態別着回数の仕様）

#### F1-4. 回り（芝左/ダ右/ダ左/ダ直 等）

* 仕様書の該当ブロックに基づいて「回り別 starts」等を作る

#### F1-5. 登録レース数

* `CK_registered_races_n`（履歴の厚み）

#### F1-6. 賞金（騎手/調教師/馬主/生産者：年/累計の合算）

* MING/CKブロック内の「本賞金・付加賞金」を合算して使う（単位はデータ定義に従う）
  （※ここは“合算して12列に圧縮”は以前合意の方針）

# 3. Market追加特徴量（Model Mのみ：0B41バックフィル前提）

ここからは **オッズありモデルに“追加”**される特徴量。

> 注意：本章は **0B41（時系列オッズ）に依存**するため、運用で使うなら **学習・検証用にも 0B41 をバックフィル**して整合させる（参照: `docs/ダウンロードリスト.md`）。

## 3A. T-5 オッズスナップ（必須）

* `M_odds_win_t5`：単勝オッズ（T-5採用レコード）
* `M_imp_prob_t5 = 1 / M_odds_win_t5`
* `M_log_odds_t5 = log(M_odds_win_t5)`
* `M_odds_rank_t5`：オッズ順位（昇順）
* `M_odds_state_flag_*`：取消/無投票など（0.4参照）

## 3B. オッズ時系列（T-60〜T-5、最大12点を推奨）

前提：0B41→O1（時系列）を使用

### 入力系列の作り方（固定）

* `snap_times = {T-60, T-55, ..., T-5}` の各時点について
  `announce_mmddhhmi <= snap_time` の最新 O1（中間）を採用
* 欠けた点は forward-fill（直前値）しつつ、欠損フラグを立てる

### 派生特徴（必須セット）

* `M_odds_slope_log`：`log(odds)` を時刻（分）で回帰した傾き
* `M_odds_volatility_log`：`std(log(odds))`
* `M_odds_min`, `M_odds_max`
* `M_odds_last_change`：直近ステップの変化率
* `M_odds_jump_flag`：`abs(last_change) > threshold`（例：15%）

## 3C. レース全体の市場状態

O1には単勝票数合計がある

* `M_win_pool_total_t5`：単勝票数合計（T-5採用O1から）
* `M_win_pool_growth_60to5`：T-60→T-5 の増分（取れる場合）
* `M_market_entropy_t5`：`imp_prob_t5` を正規化して entropy（荒れ度）

# 4. 最終オッズ予測モデル由来の特徴量（任意だが推奨）

（別モデルで `odds_close_hat` を出し、Model MやMETAに入力）

## 4A. 予測最終オッズ

* `META_odds_close_hat`
* `META_log_odds_close_hat`
* `META_odds_close_delta = META_odds_close_hat - M_odds_win_t5`
* `META_odds_close_ratio = META_odds_close_hat / M_odds_win_t5`

## 4B. EV推定の補助（意思決定用）

* 単勝の真EV：`EV = p * o - 1`
* 予測EV：`EV_hat = p * odds_close_hat - 1`
* **EV推定誤差の分解**：`ΔEV = p*(odds_close_hat - o)`
  （※この定義自体は全体仕様書で固定済み。オッズの最終/確定区分は O1の区分で管理）

# 5. 欠損・例外処理（固定）

* `"----"`, `"****"`, `"0000"`, `" "` を欠損判定対象として扱う

* `data_kbn=0` は削除通知（採用対象外）

## 5.3 CK母数0

* `starts=0` の率は 0（またはNULL）＋ `ck_no_history_flag=1`
