# ダウンロードリスト v1.0（単勝・T-5、2016〜現在）

## 1) 必須：学習・検証データ構築（JVOpen＝蓄積系）

学習データ（2016〜現在）を再現可能に作るために **最低限これだけ**を落とします。

|  優先 | dataspec（JVOpen）  | 主レコード                  | 使う特徴量（対応）                      | 理由/備考                                            |
| --: | ----------------- | ---------------------- | ------------------------------ | ------------------------------------------------ |
| ★★★ | **DIFF（マスタ差分）**    | UM / KS / CH（他混在あり）     | マスタ（馬/騎手/調教師）                   | 初回ロード推奨。FK回避・欠損補完の前提になることが多い                   |
| ★★★ | **RACE（セットアップ）**  | RA / SE / HR / O1（※）   | 過去走・レース条件・ラベル（勝敗）・確定最終オッズ（＋票数合計） | セットアップデータに RA/SE/HR/O1 が含まれる（長期収録）              |
| ★★★ | **MING（蓄積マイニング）** | DM / TM（直前時点）          | DM/TM特徴量（学習はMINGを正）            | MINGは「直前予想(馬体重発表後)時点」のDM/TMを月曜14:00頃提供           |
| ★★★ | **SNPN（出走時点情報）**  | CK（出走別着度数）             | CKミニ特徴量（着度数・脚質傾向・賞金系など）        | 出走馬名表発表時点の着度数情報として提供。JVOpenでSNPN指定可              |
| ★★☆ | **SLOP（坂路調教）**    | （坂路調教レコード）             | 調教特徴量（坂路）                      | 提供タイミング/単位が記載（毎日、不規則）                            |
| ★★☆ | **WOOD（ウッド調教）**   | WC                     | 調教特徴量（ウッド）                     | WOODは「2021/7/27以降」のウッド調教情報と記載（2016〜2021は欠損前提で設計） |
| ★☆☆ | **COMM（コース情報など）** | CS                     | コース固定情報（トラック/改修等）              | コース情報CSがCOMMに含まれる記載。なくても回るがあると便利                 |

> JVOpenで指定可能な dataspec の一覧に DIFF/RACE/MING/SNPN/SLOP/WOOD/COMM（+YSCH等）が含まれることは仕様書に明記されています。
>
> ※ RACEには環境/時期によりH1等が混在し得ますが、本プロジェクトの最小構成では **O1の「票数合計」**を利用する前提で、H1は必須にしません（必要になったら別途対応）。

### 重要：SNPNについて（あなたの補足を反映）

* **本プロジェクトでは “SNPNのみ” を使用**（SNAPは使用しない）
* あなたが「SNPNで全期間ダウンロードできることを確認済み」なので、2016〜現在は **SNPNで統一取得**でOK

  * 実装は「SNPNだけを取得し、取り込み結果で期間カバレッジを監査（欠損があればアラート）」にします。

## 2) 必須：運用当日（T-5推論）リアルタイム取得（JVRTOpen＝速報系）

運用（当日T-5）で必要な “時点固定入力” はこれです。

|  優先 | dataspec（JVRTOpen） | レコード                | 使う特徴量（対応）                            | 根拠                        |
| --: | ------------------ | ------------------- | ------------------------------------ | ------------------------- |
| ★★★ | **0B41**           | O1（時系列オッズ：単複枠）      | T-5オッズ、オッズ推移（傾き/変化/ボラ等）、最終オッズ予測モデル入力 | 0B41は「単複枠の中間オッズを時系列に収録」   |
| ★★★ | **0B11**           | WH（馬体重）             | 馬体重・増減（T-5スナップ）                      | 速報馬体重0B11→WH              |
| ★★★ | **0B16**           | WE/AV/JC/TC/CC（変更系） | 天候馬場、取消、騎手変更、発走時刻変更、コース変更            | 0B16でこれらが取得対象。イベント取得の説明あり |
| ★★☆ | **0B13**           | DM（速報タイム型）          | 当日推論用DM（kbn=3→2→1）                   | 速報DM 0B13。上書き更新の説明        |
| ★★☆ | **0B17**           | TM（速報対戦型）           | 当日推論用TM（kbn=3→2→1）                   | 速報TM 0B17。上書き更新の説明        |

> 補足：学習は **MING（蓄積）**を正にして「直前予想」を再現し、運用は **速報DM/TM**を T-5でスナップして使う、という整理は仕様書の提供形態（上書き更新/成績区分など）とも整合します。

# 4) 「このリストだけで特徴量仕様を満たせる？」チェック

あなたが確定した特徴量群に対して、上の必須セットで **満たせます**。

* **近走成績・集約・自作SR**：RACE（RA/SE）で過去走と条件が揃う
* **ラベル（1着）**：SE/HR（結果・払戻）で確定可能
* **確定最終オッズ（教師・EV検証）**：セットアップO1（確定オッズ）
* **時系列オッズ（T-60〜T-5）**：0B41（O1時系列）
* **馬体重・変更系（T-5）**：0B11/0B16
* **DM/TM**：学習はMING、運用は0B13/0B17
* **CK**：SNPN（CK）
