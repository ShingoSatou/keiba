# 検証設計仕様書（単勝・T-5・Early stopping対応・校正器なし）v1.2

## 1. 目的

1. 勝率モデルの品質（確率としての良さ含む）
2. 最終オッズ予測モデルの品質（EV推定の誤り削減）
3. 賭け方（意思決定）としての品質（ROI/分散/DD）

---

## 2. 前提（変更なし）

* 期間：2016年以降〜現在
* 推論基準：**T-5**（発走時刻の5分前）
* SNAPは使わず **SNPNで全期間**
* モデル：Model F（オッズなし）/ Model M（オッズあり）/ Model O（最終オッズ予測）/ Ensemble（stacking）

---

## 3. as-of固定とリーク防止（変更なし）

* `asof_ts = post_time - 5min`
* 各データは `<=asof_ts` の最新のみ採用
* TC等で発走時刻が変わったら asof_ts を再計算して再抽出
* O1時系列は採用したスナップのキー（announce時刻等）を監査ログに保存

---

## 4. 正解定義（変更なし）

* ラベル：`y_win = 1着なら1 else 0`
* 最終オッズ正解：`o_final` は **確定優先、なければ最終**（欠損は除外）

---

## 5. 分割設計（Early stopping前提：ここが肝）

### 5.1 外側分割（Walk-forward）

外側は「性能推定」のための検証。**OuterValidは一切学習に使わない**。

例：

* Fold1：Train=2016–2018 / OuterValid=2019
* Fold2：Train=2016–2019 / OuterValid=2020
* …（最新年をFinal Testで完全ホールドアウト）

### 5.2 内側分割（OuterTrain内の時系列3分割）

OuterTrainをさらに時系列で分割して、用途を固定する。

* **Train-fit**：モデル学習に使う
* **ES-valid**：early stopping（best iteration/epoch決定）に使う
* **Calib**：※校正器は作らないが、ここは **(a) stackingメタモデル学習** と **(b) 賭け閾値 τ 決定** に使う

推奨固定（まずはこれ）：

* `ES-valid = 90日`
* `Calib = 60日`
* `Train-fit = OuterTrainの残り`

> 重要：校正器は廃止したが、**Calibは不要ではない**です。
> stackingメタモデルと τ 決定の“汚れ仕事”をOuterValidに持ち込まないために必要です。

---

## 6. 学習手順（Fold内の標準パイプライン：校正器なし版）

### 6.1 Baseモデル（F/M）の学習（early stopping）

各モデルについて：

1. `Train-fit`で学習
2. `ES-valid`の **Logloss** を監視して early stopping
3. best checkpointで確定
4. 確定モデルで `Calib` と `OuterValid` に対して予測 `pF_raw`, `pM_raw` を出す

**early stoppingの固定パラメータ**

* `patience`：モデルごとに固定（例：100）
* `min_delta`：例 `1e-4`
* 監視指標：**Logloss（最優先）**（AUCで止めない）

### 6.2 最終オッズ予測モデル（O）の学習（early stopping）

* 目的変数：`log(o_final)` 推奨
* 監視指標：`MAE(log(o_final))`（またはHuber）
* 学習手順は同じ（Train-fit + ES-validで早期終了）
* `Calib` と `OuterValid` に `ô_close` を出す

---

## 7. Stacking（メタモデル）の仕様：ここが“校正っぽい役割”を担う

### 7.1 メタモデル学習データ（固定）

* **Calibのみ**で学習（OuterValid禁止）
* 入力例（確定）：

  * `pF_raw`, `pM_raw`
  * `ô_close`（または `log(ô_close)`）
  * （必要なら）`odds_t5` や `log_odds_t5` を少数追加
* 目的：`y_win`

### 7.2 メタモデルの型（推奨）

* 第一候補：**ロジスティック回帰（L2）**

  * これが一番「確率っぽく」なりやすい（=校正寄り）
* 第二候補：LightGBM等（ただし確率が歪むことがあるので、まずはLR推奨）

### 7.3 出力（校正器なし）

* `p_final = meta_model(pF_raw, pM_raw, ô_close, …)`
* これが **そのまま**意思決定と評価に使う確率

---

## 8. 賭け閾値 τ の決め方（Calibで固定、OuterValidで固定適用）

### 8.1 EV定義（純利益ベース）

* 推定EV：`EV_hat = p_final * ô_close - 1`
* 真のEV（検証用）：`EV_true = p_final * o_final - 1`

  * ※ここでは「確率の真値」は不明なので、**“推定確率に対して最終オッズ差がどれだけ効いたか”**を見る意味合い

### 8.2 τ決定（固定ルール）

* τは **Calibのみで**探索し固定
* 目的：ROI最大（ただし購入回数が極端に少ない解は除外）
* 追加制約（推奨）：最大DDが許容内

### 8.3 ベットルール（標準）

* 1レース最大1頭（EV_hat最大）
* 購入条件：`EV_hat >= τ`
* 賭け金：固定100円
* 取消・返還：profit=0

---

## 9. 評価指標（校正器は無しでも“校正指標は監視として残す”）

### 9.1 モデル品質

* AUC
* Logloss（最優先）
* Brier

### 9.2 校正の診断（校正器は使わないが、悪化監視のため必須）

* Reliability diagram
* ECE（Expected Calibration Error）

> ここで「校正が崩れていてEVが不安定」な兆候が出たら、次フェーズで校正器導入を検討（今は入れない方針でOK）。

### 9.3 最終オッズ予測

* `MAE/RMSE(log(o_final) - log(ô_close))`
* 意思決定一致率：`sign(p*ô-1) == sign(p*o-1)`（pはp_final）

### 9.4 運用指標（バックテスト）

* ROI、購入回数、的中率
* 最大DD
* 月次/開催別分解
