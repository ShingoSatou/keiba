# スクリプトリファレンス（コマンド・用途一覧）

`scripts/` 以下のファイル群の役割と代表的な実行コマンドを工程別に完全網羅したリファレンスです。
不要ファイルや旧実装（MVP用）と、現在の本線パイプライン（`_t5`系等）を明確に区別しています。

---

## 1. データ取得系（JV-Link → JSONL）

| スクリプト | 役割と用途 | 実行例 |
|---|---|---|
| `extract_jvlink.py` | **【蓄積系】通常データ取得**<br>JVOpen経由で過去レースデータを全件取得する。 | `.\.venv32\Scripts\python.exe scripts/extract_jvlink.py --spec RACE --from 20160101` |
| `extract_rt_jvlink.py` | **【速報系】リアルタイム単発取得**<br>JVRTOpen経由で指定dataspec単発の当日データを取得。 | `.\.venv32\Scripts\python.exe scripts/extract_rt_jvlink.py --spec 0B14 --key 20260215050106` |
| `backfill_rt.py` | **【蓄積系】速報のバックフィル取得**<br>0B41等、racekey単位での過去遡及取得用バッチ。 | `.\.venv32\Scripts\python.exe scripts/backfill_rt.py --spec 0B41 --from-date 2016-01-01` |
| `ops_rt.py` | **【運用系】当日一括取得**<br>運用当日朝やT-5に必要な全dataspec(0B41/0B14/0B11等)を一括取得。 | `.\.venv32\Scripts\python.exe scripts/ops_rt.py --race-date 20260215` |
| `rt_common.py` | **【共通群】**<br>WSLからWindows側の32bit Pythonを透過的に呼び出すユーティリティ。単体では叩かない。 | - |

---

## 2. DB構築・投入系（JSONL → PostgreSQL）

| スクリプト | 役割と用途 | 実行例 |
|---|---|---|
| `migrate.py` | **DBのDDL適用初期化**<br>`migrations/*.sql`を使用してスキーマを構築する。 | `uv run python scripts/migrate.py --db postgres://jv...` |
| `verify_parsers.py` | **パーサ検証**<br>DBに入れる前にJSONLファイルのレコードがパースできるかチェック。 | `uv run python scripts/verify_parsers.py` |
| `load_to_db.py` | **【主軸】DBバッチ投入**<br>JSONLファイルを読み込み、`raw`レイヤと`core`レイヤに冪等で挿入する。 | `uv run python scripts/load_to_db.py --input "data/RACE_*.jsonl"` |
| `ops_rt_load.py` | **【運用系】当日速報のDB投入**<br>当日のリアルタイムデータ（0B*系）をDBに投入する専用ラッパー。 | `uv run python scripts/ops_rt_load.py --race-date 20260215` |
| `ops_race.py` | **【運用系】当日出走表の抽出・投入**<br>運用当日のRACEデータ（出走表）だけをDBに投入して枠を作る。 | `uv run python scripts/ops_race.py --race-date 20260215` |

---

## 3. 特徴量・スナップショット系（Core → Mart）

ここから機械学習テーブルの生成です。

| スクリプト | 役割と用途 | 実行例 |
|---|---|---|
| `build_features.py` | **【主軸】基礎特徴量の生成**<br>Core層から馬単位・血統・騎手実績等を計算し`mart`層の各種statsに永続化。 | `uv run python scripts/build_features.py` |
| `build_t5_snapshot.py` | **【運用互換】T-5時点スナップショット生成**<br>運用当日の「T-5（発走5分前）までの最新」固定で全入力データ（オッズ含む）を生成し`mart.t5_runner_snapshot`に格納。 | `uv run python scripts/build_t5_snapshot.py --date 20260215` |
| `market_features.py` | **【共通群】オッズ推移特徴量計算関数**<br>確定前オッズからボラティリティなどの特徴量を計算する。単体では叩かない。 | - |

---

## 4. 学習・MLパイプライン系（Mart → Model）

旧MVP系（v0.1）と、現在の本線であるT-5完全互換（v0.2）アーキテクチャに分かれます。

| スクリプト | 役割と用途 | 状態 | 実行例 |
|---|---|---|---|
| `build_dataset_t5.py` | **【本線】Walk-Forward用データ生成**<br>T-5スナップと特徴量を結合し、時系列オッズを含むLightGBM用データセットを生成。 | ✅ 現行主軸 | `uv run python scripts/build_dataset_t5.py` |
| `train_walk_forward.py` | **【本線】時系列交差検証（CV）**<br>Model F / M / O / Stacking の性能をリークのないWalk-Forwardで評価・最適化。 | ✅ 現行主軸 | `uv run python scripts/train_walk_forward.py` |
| `train_production_bundle.py` | **【本線】本番デプロイモデル生成**<br>設定された運用期間までの全データを使ってアンサンブルモデルをコンパイル。 | ✅ 現行主軸 | `uv run python scripts/train_production_bundle.py` |
| `check_dataset_quality.py` | **【共通群】品質テスト**<br>生成された `data/train.parquet` にバグ列やNULLがないかバリデーション。 | ✅ 現行主軸 | `uv run python scripts/check_dataset_quality.py` |
| `t5_modeling.py` | **【共通群】モデル定義**<br>Model F/M/Oの特徴量定義とStackingロジック。 | - | - |

---

## 5. 推論・運用テスト系（Model → Decision）

| スクリプト | 役割と用途 | 状態 | 実行例 |
|---|---|---|---|
| `ops_t5_dryrun.py` | **【本線】T-5本番推論オーケストレータ**<br>取得→DB→特徴量→スナップ→推論→買い目表出力までを一気通貫で実行。 | ✅ 現行主軸 | `uv run python scripts/ops_t5_dryrun.py --race-date 20260215` |
| `predict_t5.py` | **【本線】モデルスタッキング推論器**<br>`ops_t5_dryrun.py`内で呼ばれ、T-5スナップとBundleモデルからEV買い目JSONを出力。 | ✅ 現行主軸 | `uv run python scripts/predict_t5.py` |
| `backtest_v2.py` | **【本線】運用バックテスト**<br>T-5スナップとCV予測値を用いて、過去に「買っていたら資産がどう推移したか」をシミュレート。 | ✅ 現行主軸 | `uv run python scripts/backtest_v2.py` |
| `optimize_thresholds.py` | **【運用系】最適閾値探索**<br>バックテスト出力をもとに、EV閾値等のベターなパラメータを探す。 | ✅ 現行主軸 | `uv run python scripts/optimize_thresholds.py` |
