## **1\. プロジェクトの目的**

市場の非効率性を突き、長期的かつ安定的な利益（回収率100%超え）を創出する完全自動化された競馬予測・投票システムの構築。単純な的中率ではなく、リスクを統制した上での「期待値の最大化」を目的とする。

## **2\. ターゲットの定義（馬券種と運用方針）**

* **対象馬券種:** **ワイド**（順不同で3着以内に入る2頭を的中させる）  
* **選定理由:** 1着固定によるノイズ（出遅れ等）を排除し、不確実性をコントロールしつつ、複勝よりも高いオッズの歪み（期待値）を狙うため。  
* **購入戦略:** モデルが算出する入着確率と直前オッズから期待値を計算し、複数点買いによる分散投資を行う。

## **3\. モデルの課題設定（Problem Formulation）**

* **アプローチ:** **ランキング学習（Learning-to-Rank） / Listwiseアプローチ**  
* **アルゴリズム候補:** LightGBM または XGBoost の LambdaMART実装  
* **データ構造:** 縦持ち（1行＝1頭）とし、レースID（Query ID / Group）をキーとしてレースごとのリストを構成する。

### **3.1. 正解ラベル（Relevance Label）の設計**

ワイド馬券の的中条件に合致させるため、上位3頭に対して非線形なスコアを付与する。

* 1着: 3  
* 2着: 2  
* 3着: 1  
* 4着以下: 0

### **3.2. 評価指標（Evaluation Metrics）**

* **オフライン評価指標:** **NDCG@3** (Normalized Discounted Cumulative Gain at rank 3\)  
* **選定理由:** 4着以下の順位予測エラーを無視し、「上位3頭に食い込む確率が高い馬」を正確にリスト上位へランク付けする能力のみを評価するため。

## **4\. 予測対象レースのセグメント（初期フェーズ）**

不確実性の高い要素を排除し、データの質が担保されたレースのみにスコープを絞る。

* **対象馬場:** **ダートのみ**（天候やトラックバイアスによるノイズが芝に比べて少ないため）  
* **対象クラス:** **4歳以上の条件戦およびオープン・重賞**  
* **除外対象:**  
  * 障害戦（競技性質が異なるため）  
  * 芝レース（ダートと要求される適性が異なるため）  
  * 新馬戦・未勝利戦（過去のパフォーマンスデータが不足しているため）

## **5\. 推論実行タイミングとデータ要件**

* **実行タイミング:** **レース発走の5分前**  
* **利用データ:**  
  * 前日までに確定している履歴データ（過去走、血統、調教など）  
  * 当日発走5分前までに取得可能な直前データ（速報馬体重、天候・馬場状態の変更、騎手変更、**発走5分前時点の速報オッズ**）  
* **データソース:** JRA-VAN Data Lab. (JV-Linkを利用したAPI取得)

## **6\. 特徴量設計における絶対遵守事項**

1. **データリークの完全排除:**  
   * バックテスト（過去データでの検証）において、「確定オッズ」「最終人気」「レース確定後の馬場状態」など、発走5分前時点で絶対に知り得ない未来の情報を特徴量に1ミリでも混入させてはならない。  
2. **レース内相対評価の明示的付与:**  
   * モデルの入力データは「1行＝1頭」であるため、各馬の絶対値データ（走破タイムなど）だけでなく、必ず「該当レースに出走するメンバー内での偏差値」や「ランク（順位）」に変換した相対評価特徴量を生成し、モデルに入力すること。

## **7\. リスク管理（オッズ・ドロップ対策）**

発走5分前のオッズは確定オッズではない。大口投票による直前のオッズ下落（オッズ・ドロップ）によって、購入時の期待値がレース確定後に1.0を下回る致命的リスクが存在する。

* **要件:** 発走5分前オッズと確定オッズの過去の変動履歴（速報票数データ）を分析し、オッズ下落の分散をリスクとして組み込んだポートフォリオ最適化（ケリー基準のフラクショナル適用など）を実装すること。
