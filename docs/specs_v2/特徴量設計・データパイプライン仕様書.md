# **競馬予測システム：特徴量設計・データパイプライン仕様書**

## **1\. 概要**

本ドキュメントは、JV-Link（蓄積系および速報系データ）から抽出した情報を元に、ランキング学習（Listwise / NDCG@3最適化）に入力するための特徴量マトリクスを生成する手順と計算ロジックを定義する。

1行が「1レースの出走馬1頭」に対応し、同レース内の馬群はRace\_ID（Query ID）で束ねられる。

**※データの品質と網羅性を担保するため、対象は「中央競馬（JRA）」のみに限定する。**

## **2\. 利用するJV-Dataレコード種別**

特徴量生成の基盤となるテーブル群。各テーブルはRace\_ID（開催年＋月＋日＋場コード＋回次＋日次＋レース番号）および馬コードをキーとして結合する。

1. **レース詳細 (RA)**: レースの条件（距離、馬場、クラス、頭数）および結果。  
   * **【抽出条件】場コードが 01〜10（札幌〜小倉のJRA全10場）のレースのみを対象とする。地方競馬・海外レースのデータはパイプラインの入り口で完全に除外すること。**  
2. **馬毎レース情報 (SE)**: 各馬のレース結果詳細。  
3. **競走馬マスタ (UM)**: 血統、生年月日、性別。  
4. **調教師 (CH) / 騎手 (KS) マスタ**: 所属、リーディング実績の算出基盤。  
5. **速報タイム型データマイニング予想 (DM \- 0B13)**: JRA-VAN公式のタイムベース予測値。  
6. **速報対戦型データマイニング予想 (TM \- 0B17)**: JRA-VAN公式の対戦ベース予測値。

## **3\. 特徴量群（Feature Groups）の定義**

モデルに学習させる特徴量は、以下の6グループに分類して生成する。

### **Group A: 基礎属性・条件特徴量（Absolute Context）**

レースの前提条件と、対象馬の不変的な属性。

* age: 年齢（レース開催日 \- 生年月日より算出）  
* sex: 性別（牡/牝/センのOne-Hotエンコーディング）  
* weight\_carried (斤量): 負担重量（kg）。  
* distance: レース距離（m）。  
* track\_condition: 馬場状態（良、稍重、重、不良）。※直前予測モデル用。  
* is\_jockey\_change: 乗り替わりフラグ（前走と同じ騎手か否か。1 or 0）。  
* jockey\_id / trainer\_id: 騎手・調教師のコード。LightGBMのcategorical\_featureとして指定する（※過学習を防ぐため、出現頻度の低いIDは「その他(99999)」に丸める頻度エンコーディングの適用を推奨）。

### **Group B: 過去走パフォーマンス特徴量（Lag Historical Performance）**

対象馬の過去のレース成績。異条件の単純平均によるノイズを防ぐため、直近3走（Lag1〜Lag3）を展開。

* lag1\_speed\_index: 前走のスピード指数。  
  * **※計算定義:** (基準タイム \- 自身の走破タイム) \* 距離補正 \+ 80  
  * **※基準タイム:** 過去の基本基準タイム ＋ 当日の馬場差（全レースのズレ幅平均）  
  * **※距離補正:** （任意の基準距離 例:1440m） ÷ （そのレースの距離）  
* lag1\_up3\_index: 前走の上がり3F指数。  
* lag1\_distance\_diff: 今回のレース距離と前走の距離の差分。  
* lag1\_course\_type\_match: 前走のトラック種別（芝/ダート）が今回と同じか否か（1 or 0）。  
* lag1\_rank: 前走の着順。  
* lag2\_..., lag3\_...: 2走前、3走前の同上データ。  
* days\_since\_lag1: 今回のレースから前走（Lag1）までの休養日数。

### **Group C: コース・条件適性特徴量（Specific Aptitude）**

今回のレース条件と同一、または類似する条件における過去の成績。

* aptitude\_same\_distance\_win\_rate: 過去2年間の同距離での複勝率（ベイズ平滑化必須）。  
* aptitude\_same\_condition\_win\_rate: 過去2年間の同一馬場状態での複勝率（ベイズ平滑化必須）。

### **Group D: レース内相対評価特徴量（Listwise Relative Features）★最重要★**

Group A, B, C, Fの絶対値を、**「今回の出走メンバー内での偏差値」または「ランク（順位）」に変換**した特徴量。

* rel\_lag1\_speed\_index\_z: lag1\_speed\_indexのレース内偏差値（Z-score）。  
* rel\_aptitude\_distance\_z: 同距離実績のレース内偏差値。  
* rel\_weight\_carried\_z: 斤量のレース内偏差値。  
* rel\_jockey\_win\_rate\_z: 騎手勝率（Group F）のレース内偏差値。「今回の出走メンバーの中で、相対的にどれくらい腕の良い騎手が乗っているか」をモデルに認識させる。

### **Group E: 外部メタ特徴量（External Meta Features）**

JRA-VAN公式が提供するデータマイニング予測値（事前知識）。

* meta\_dm\_predict\_time: タイム型予測走破タイム（連続値）。  
* meta\_dm\_predict\_rank: タイム型予測順位（順序尺度）。  
* meta\_tm\_predict\_score: 対戦型予測スコア（連続値）。  
* meta\_tm\_predict\_rank: 対戦型予測順位（順序尺度）。  
* rel\_meta\_dm\_time\_z: 予測走破タイムのレース内偏差値。

### **Group F: 陣営評価特徴量（Connections Performance）**

騎手および調教師の能力を定量化した特徴量。データリークを完全に防ぐため、集計には細心の注意を払う。

* jockey\_recent\_6m\_win\_rate: 該当レースの**前日時点**から遡って過去6ヶ月間における、該当騎手の複勝率。  
* trainer\_recent\_6m\_win\_rate: 該当レースの**前日時点**から遡って過去6ヶ月間における、該当調教師の複勝率。  
  * **※計算要件:** リーク防止のため、必ず時系列順にデータを処理し、「現在処理中のレース結果」が含まれないようにExpanding Window（またはRolling Window）で集計すること。試行回数が少ない（例: 過去半年で5走のみ）場合は、全体の平均勝率に寄せるベイズ平滑化を必ず行う。

## **4\. データ処理とリーク防止の厳格なルール**

1. **情報のタイムスタンプ制約:**  
   特徴量計算に用いるデータは、全て「対象レースの発走日時（![][image1]）より前に確定しているデータ」でなければならない。  
2. **「確定オッズ」および「最終人気」の除外:**  
   レース結果テーブル（RA, SE）に含まれる確定単勝オッズ、確定単勝人気を特徴量として用いることは**固く禁ずる**。  
3. **ターゲット・エンコーディング（Target Encoding）の罠:**  
   騎手・調教師勝率（Group F）を計算する際、計算対象のレース結果を含めて集計してはならない。Pandasのgroupby().transform('mean')のような全体集計は即座にリークを引き起こす。  
4. **欠損値（NaN）の取り扱い方針:**  
   過去走が少ない馬、あるいは**地方・海外からの転入馬**などの欠損値はそのまま（np.nan / None）モデルに投入し、ツリーアルゴリズムのNaN最適分岐に委ねる。

## **5\. 出力データのフォーマットとソート要件**

前処理パイプラインの最終出力。

**【重要】出力データは必ず Race\_ID で昇順にソートされていなければならない。**

| カラム名 | データ型 | 説明 |
| :---- | :---- | :---- |
| Race\_ID | String | クエリID。LambdaMARTのグループ指定に使用。**※この列でソート必須** |
| Horse\_ID | String | 馬コード。 |
| Target\_Label | Integer | NDCG評価用の正解ラベル（1着=3, 2着=2, 3着=1, 4着以下=0）。 |
| feature\_age | Float | 特徴量（Group A） |
| feature\_lag1\_speed\_index | Float | 前走のスピード指数（Group B）。 |
| feature\_jockey\_recent\_win\_rate | Float | 騎手の直近6ヶ月勝率（Group F）。※厳密なローリング集計必須 |
| feature\_rel\_jockey\_win\_rate\_z | Float | 騎手勝率のレース内偏差値（Group D）。 |
| feature\_meta\_tm\_rank | Integer | 公式対戦型マイニング予測順位（Group E） |
| feature\_rel\_lag1\_speed\_z | Float | 前走スピード指数のレース内偏差値（Group D）。 |
