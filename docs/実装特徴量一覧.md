## 1. ベースライン学習データ `data/train.parquet`

生成スクリプト: `scripts/build_dataset.py`

### 1.1 抽出条件（母集団フィルタ）

`core.race` / `core.runner` / `core.result` を結合し、以下の条件で抽出します。

* `track_code BETWEEN 1 AND 10`（中央）
* `surface IN (1,2)` がデフォルト（`--include-obstacles` で `(1,2,3)`）
* `distance_m > 0`
* `run.scratch_flag = FALSE`
* `res.finish_pos IS NOT NULL`
* 「勝ち馬が存在するレースのみ」: `EXISTS (finish_pos=1)`
* 生成後に `--min-horses` で少頭数レースを除外（デフォルト5）

### 1.2 出力列（全体）

`train.parquet` は概ね次の列を持ちます（CKが取れている前提で約123列）。

* メタ:
  * `race_id`, `race_date`, `horse_id`, `horse_no`
* レース条件（`core.race`）:
  * `track_code`, `surface`, `distance_m`, `going`, `class_code`, `field_size`
* 出走馬（`core.runner`）:
  * `gate`, `carried_weight`, `body_weight`, `body_weight_diff`
* 当日コンディション（算出）:
  * `days_since_last`, `distance_change_m`
* 近走集約（`mart.horse_stats`）:
  * `speed_*`, `closing_*`, `early_*`, `position_gain_*`, `finish_*`, `n_runs_5`
  * 条件近似版: `*_sim_*`, `n_sim_runs_5`（ただしベースライン入力からは除外中）
* レース内相対（算出）:
  * `*_z_inrace`, `*_rank`
* ペース圧（算出）:
  * `pace_front_runner_cnt`, `num_faster_early`
* 人の実績（`mart.person_stats`）:
  * `jockey_win_rate_1y`, `jockey_place_rate_1y`, `trainer_win_rate_1y`, `trainer_place_rate_1y`
* DM/TM（`core.mining_dm` / `core.mining_tm`）:
  * `dm_*`, `tm_*`（欠損フラグ含む）
* 調教（`core.training_slop` / `core.training_wood`）:
  * `slop_*`（ベースライン入力に含む）
  * `wood_*`（列は出るがベースライン入力からは除外中）
* CK（`mart.feat_ck_win`）:
  * `ck_*`（45列 + 欠損フラグ + 平滑化率）
* 目的変数:
  * `is_win`（= `finish_pos==1`）, `finish_pos`

### 1.3 値の補正（型/欠損の取り扱い）

`build_dataset.py` の保存前に以下の補正を入れています。

* `class_code`: `NULL→0` に統一し `int` 化
* `going`: `0→NULL`（欠損）に変換
* `field_size`: `0→NULL`（欠損）に変換

## 2. 各特徴量の算出方法（実装ベース）

ここでは **列名（実装）**を正として、どこから/どう作るかをまとめます。

### 2.1 レース条件（`core.race`）

* `track_code`: 競馬場コード（01-10）
* `surface`: 1=芝,2=ダ,3=障害（デフォルトで3は除外）
* `distance_m`: 距離(m)
* `going`: 馬場状態（`0` は欠損化）
* `field_size`: 頭数（`0` は欠損化）
* `class_code`: **現状は未実装で `0` 固定になりがち**（`NULL→0`）。仕様上は「条件/格」を入れる枠だが、いまは情報量ゼロ。

### 2.2 出走馬（`core.runner`）

* `gate`: 枠番
* `carried_weight`: 斤量
* `body_weight`, `body_weight_diff`: 馬体重/増減（欠損はそのまま）

### 2.3 当日コンディション（`_add_condition_features`）

過去の出走履歴（`core.runner`×`core.race`）から `LAG()` で前走を取り、当該 `race_date` に結合します。

* `days_since_last = race_date - prev_race_date`（日数）
* `distance_change_m = distance_m - prev_distance_m`
  * `prev_distance_m` が無い場合は `distance_change_m = 0` になる（`prev_distance_m` を当該距離で補完）

### 2.4 走行指数の元（`mart.run_index` / `scripts/build_features.py` Step 2）

`mart.run_index` は「1走=1行」の基礎指標です（`core.result` を入力）。

* `distance_bucket`: `distance_m` を上限バケットへ丸め（1000,1200,…,3600）
* `going_bucket`: `going` を2値化（良系=1、道悪系=2）
* `speed_index`:
  * グループ: `track_code,surface,distance_bucket,(going_bucket)`
  * `time_sec` の **指数移動平均（EWM）**でベンチマーク平均/標準偏差を推定し、`Shift(1)` してリーク防止（当該走の情報を使わない）
  * Fine（goingあり）と Coarse（goingなし）を **縮約（shrinkage）**で混合
    * `w = n / (n + k)`（`n=count`, `k=20`）
    * `mean = w*mean_fine + (1-w)*mean_coarse`（欠損は coarse で補完）
    * `std = w*std_fine + (1-w)*std_coarse`
  * `speed_index = (bm_mean - time_sec) / bm_std`
* `closing_index`: `final3f_sec` を `time_sec` と同様に扱い
  * `closing_index = (bm_3f_mean - final3f_sec) / bm_3f_std`
  * `closing_missing = final3f_sec IS NULL`
* `early_index = -corner4_pos`（4角通過順を簡易に「先行力」へ変換、欠損は `corner4_pos=10` 扱い）
* `position_gain = corner4_pos - finish_pos`（4角→ゴールで上げた着順）

> 仕様書/カタログの「SR（パータイム＋バリアント）」とは別物で、現状は **EWMベンチマーク由来の標準化指数**です。

### 2.5 近走集約（`mart.horse_stats` / `scripts/build_features.py` Step 3）

対象馬（当日出走馬）ごとに、当日より前の **直近5走**を `mart.run_index` と `core.result` から取得し集約します。

* 全条件版（`past_runs`=直近5走）:
  * `speed_last`: 直近走の `speed_index`
  * `speed_mean_3`: 直近3走の平均
  * `speed_best_5`: 直近5走の最大
  * `speed_std_5`: 直近5走の標準偏差（`ddof=1`、2件未満なら欠損）
  * `speed_trend_3`: 直近3走の傾き（線形回帰 slope、2件未満なら欠損）
  * `closing_last/mean_3/best_5`: `closing_index` の同様集約
  * `early_mean_3/early_best_5`: `early_index` の同様集約
  * `position_gain_mean_3`: `position_gain` の直近3走平均
  * `finish_mean_3`: `finish_pos` の直近3走平均
  * `finish_best_5`: `finish_pos` の直近5走最小（=最高着順）
  * `n_runs_5`: 取得できた過去走数（最大5）

* 条件近似版（`sim_runs`）:
  * 条件:
    * `surface` 一致
    * `abs(past_distance_m - distance_bucket) <= 200`
    * `going_bucket` 一致（良系/道悪系）
  * `speed_sim_mean_3`, `speed_sim_best_5`, `closing_sim_*`, `early_sim_mean_3`, `n_sim_runs_5`

> 現行ベースラインでは欠損率が高いため、`*_sim_*` と `n_sim_runs_5` は学習入力（`scripts/train.py FEATURE_COLS`）から除外中です。

### 2.6 レース内相対（`_add_inrace_features`）

同一レース内で、次の3列を相対化します。

* 対象: `speed_best_5`, `closing_best_5`, `early_mean_3`
* z-score:
  * `{col}_z_inrace = (col - mean_race(col)) / std_race(col)`（`std=0` は1扱い）
* rank:
  * `{col}_rank`（降順、`method=min`、1が最強）

### 2.7 ペース圧（`_add_pace_features`）

* `pace_front_runner_cnt`: レース内で `early_mean_3_rank <= 3` の頭数
* `num_faster_early`: `early_mean_3_rank - 1`（自分より前に行ける馬の数の近似）

### 2.8 人の実績（`mart.person_stats` / `scripts/build_features.py` Step 4）

騎手/調教師ごとに、当日（`calc_date`）より前の **過去365日**を集計して保存します（当日は含めない）。

* `win_rate_1y = wins / total`
* `place_rate_1y = places / total`（`places = finish_pos<=3`）
* 学習データ側は、`(race_date, jockey_id)` / `(race_date, trainer_id)` で結合

### 2.9 DM/TM（`_join_mining_features`）

`core.mining_dm` / `core.mining_tm` を `(race_id, horse_no)` で結合します。

* `dm_data_kbn`, `dm_rank`（数値化）
* `dm_pred_time_sec = dm_time_x10 / 10.0`
* `dm_missing_flag = isna(dm_pred_time_sec)`
* `tm_data_kbn`, `tm_rank`（数値化）
* `tm_score = tm_score / 10.0`
* `tm_missing_flag = isna(tm_score)`

> 仕様書の「T-5以前の最新を採用（as-of）」とは別で、現状は **MING（蓄積）を単純結合**しています。

### 2.10 調教（`_join_training_features`）

対象: `core.training_slop`（坂路）, `core.training_wood`（ウッド）

* 「最終追い切り」: `training_date <= race_date` かつ `race_date-60日 <= training_date` の範囲で
  * `ORDER BY training_date DESC, training_time DESC` の先頭1件を採用
* `*_days_since_last = race_date - training_date`（無ければ欠損）
* `*_count_28d`: `race_date-28日..race_date` の件数（無い場合は `0`）
* 欠損フラグ:
  * `slop_missing_flag = isna(slop_last_total_4f_sec)`
  * `wood_missing_flag = isna(wood_last_total_4f_sec)`

> 現行ベースラインでは `wood_*` は学習入力から除外中（列は `train.parquet` に出ます）。

### 2.11 CK（`_join_ck_features`）

`mart.feat_ck_win` を race_id に変換して `(race_id, horse_id)` で結合します。

* `race_id` 変換:
  * `race_id = (kaisai_year*10000 + kaisai_md)*10000 + track_cd*100 + race_no`
* 同一 `(race_id, horse_id)` に複数レコードがある場合:
  * `dataspec='SNPN'` を優先し、次に `data_create_ymd` が新しいものを採用
* 列名:
  * `CK_FEATURE_COLUMNS` を `ck_{col}` に rename（45列）
  * `ck_missing_flag = isna(ck_h_total_starts)`
* 平滑化率（固定）:
  * `ck_overall_win_rate_sm = (ck_h_total_wins + 1) / (ck_h_total_starts + 10)`
  * `ck_overall_top3_rate_sm = (ck_h_total_top3 + 1) / (ck_h_total_starts + 10)`
  * `ck_overall_top5_rate_sm = (ck_h_total_top5 + 1) / (ck_h_total_starts + 10)`
  * `ck_central_win_rate_sm = (ck_h_central_wins + 1) / (ck_h_central_starts + 10)`
  * `ck_central_top3_rate_sm = (ck_h_central_top3 + 1) / (ck_h_central_starts + 10)`

## 4. 仕様/カタログとの乖離サマリ（大きいもの）

### 4.1 「カタログにあるが、実装/学習入力に無い（または未実装）」

* `F_grade_class`（重賞格/条件）:
  * 実装には `class_code` があるが、パース未実装で定数化しているため「格/条件」として機能していない
* `F_track_turn` / `F_post_time` / `F_weather` などのレース条件ブロック:
  * `core.race` に列はあるが（`turn_dir`, `start_time`, `weather`）、`train.parquet` に出していない/未実装
* 「直近N走の生系列」: `F_last1_*`〜`F_last5_*`
  * 現状は生系列は持たず、`mart.horse_stats` の集約のみ
* Market（`M_odds_*` 等）:
  * ベースライン学習（`train.py`）は未対応（T-5スナップは別テーブル `mart.t5_runner_snapshot` を別パイプラインで生成）

### 4.2 「実装はあるが、仕様の意図とズレがある」

* SR（速度指標）:
  * 仕様: パータイム＋トラックバリアント等で設計
  * 実装: `EWMベンチマーク` から標準化した `speed_index/closing_index` を作り、その集約を学習入力にしている
* as-of（T-5）:
  * 仕様: `<=T-5` の最新採用
  * 実装（学習データ）: `core.mining_*`（MING）等を単純結合しており、T-5キー/監査は学習データには持っていない

## 5. T-5 as-of スナップ（`mart.t5_runner_snapshot`）

`mart.t5_runner_snapshot` は **当日入力をT-5（as-of）で再現**するためのテーブルです。  
生成スクリプト: `scripts/build_t5_snapshot.py`（DDL: `migrations/0002_init_mart.sql`）

### 5.1 主な列（抜粋）

* キー/メタ:
  * `race_id`, `race_date`, `track_code`, `race_no`, `horse_id`, `horse_no`
  * `post_time`（発走時刻T）, `asof_ts`（T-5）, `feature_set`, `code_version`
* 当日情報（as-of）:
  * `gate`, `jockey_id`, `trainer_id`, `carried_weight`
  * `body_weight_asof`, `body_weight_diff_asof`, `bw_source`（`WH` or `SE`）
* オッズ（as-of）:
  * `o1_data_kbn`, `o1_announce_mmddhhmi`, `odds_win_t5`, `pop_win_t5`
  * `odds_rank_t5`, `win_pool_total_100yen_t5`, `odds_snapshot_age_sec`, `odds_missing_flag`
  * `odds_win_final`, `pop_win_final`（後日検証/学習用）
* 監査キー（JSON）:
  * `event_change_keys`（TC/JC/AV/WE/CC 等の採用キーを格納）

### 5.2 算出/採用ルール（実装の要点）

* `asof_ts = post_time - 5分`
* 体重:
  * `WH`（速報馬体重）が `<=asof_ts` で取れればそれを採用し、なければ `SE`（出馬表）の体重へフォールバック
  * `bw_source` で採用元を保持
* オッズ（単勝）:
  * `core.o1_header`（O1ヘッダ）から `announce_mmddhhmi <= asof_ts` の最新を採用
  * `core.o1_win`（馬別）から `win_odds_x10` を取り、`<=0` や `NULL` は欠損扱いにして `odds_missing_flag` を立てる
  * `odds_rank_t5` はレース内で `odds_win_t5` 昇順の `RANK()`（同率同順位）
* 変更系（0B14）:
  * `core.event_change` から `<=asof_ts` の最新を種別ごとに採用し、`event_change_keys` に格納
  * `JC` があれば `jockey_id` / `carried_weight` を置換（as-of）

> このテーブルはカタログ（T-5 as-of）に近い責務ですが、**ベースライン学習（`scripts/train.py`）の入力には未接続**です。
