# スクリプトリファレンス（v3）

## 前提

- `uv sync`
- 追加依存（必要に応じて）:
  - `uv sync --extra xgboost --extra catboost`
  - `uv sync --extra torch`
- `.env` に `DATABASE_URL` を設定

## 1. v3特徴量生成

```bash
uv run python scripts_v3/build_features_v3.py \
  --input data/features_v2.parquet \
  --output data/features_v3.parquet \
  --meta-output data/features_v3_meta.json
```

## 2. 単勝分類（OOF + model + metrics）

```bash
uv run python scripts_v3/train_win_lgbm_v3.py
uv run python scripts_v3/train_win_xgb_v3.py
uv run python scripts_v3/train_win_cat_v3.py
```

主出力（例）:
- `data/oof/win_lgbm_oof.parquet`
- `data/oof/win_lgbm_cv_metrics.json`（Benter R*を含む）
- `models/win_lgbm_v3.txt`
- `models/win_lgbm_all_years_v3.txt`
- `models/win_lgbm_bundle_meta_v3.json`

## 3. 複勝分類（OOF + model + metrics）

```bash
uv run python scripts_v3/train_place_lgbm_v3.py
uv run python scripts_v3/train_place_xgb_v3.py
uv run python scripts_v3/train_place_cat_v3.py
```

## 4. odds校正（任意）

```bash
uv run python scripts_v3/train_odds_calibrator_v3.py \
  --input data/features_v3.parquet \
  --score-cols p_win_odds_t10_norm,p_win_odds_final_norm \
  --methods logreg,isotonic
```

## 5. PL学習（uなし）

```bash
uv run python scripts_v3/train_pl_v3.py \
  --features-input data/features_v3.parquet \
  --train-window-years 3 \
  --holdout-year 2025
```

主出力:
- `data/oof/pl_v3_oof.parquet`
- `data/oof/pl_v3_cv_metrics.json`
- `models/pl_v3_recent_window.joblib`
- `models/pl_v3_all_years.joblib`
- `models/pl_v3_bundle_meta.json`

`--emit-wide-oof` 指定で `data/oof/pl_v3_wide_oof.parquet` も出力。

## 6. 1レース推論（運用=t10）

```bash
uv run python scripts_v3/predict_race_v3.py \
  --input data/inference/race_features.parquet \
  --pl-model models/pl_v3_recent_window.joblib \
  --output data/predictions/race_v3_pred.parquet \
  --emit-wide \
  --wide-output data/predictions/race_v3_wide.parquet
```

補足:
- 入力は1レースのみ（`race_id` が1種類）を想定
- 既定では base model meta から `p_win_*` / `p_place_*` を生成
- `--skip-base-model-inference` で既計算列を利用可能
- 運用パスでは final odds 依存のPLモデルを拒否（t10専用）

## 7. ワイドROI評価（v2 backtest器で算出）

v3の `p_top3` を入力として、v2の `scripts_v2/backtest_wide_v2.py` でROIを算出できる。

OOF（例: 2023–2024, `pl_v3_oof_w2.parquet`）:

```bash
uv run python scripts_v2/backtest_wide_v2.py \
  --input data/oof/pl_v3_oof_w2.parquet \
  --years 2023,2024 \
  --holdout-year 2025 \
  --output data/backtest_v3/backtest_wide_v3_pl_w2_2023_2024.json \
  --meta-output data/backtest_v3/backtest_wide_v3_pl_w2_2023_2024_meta.json \
  --force
```

`min_p_wide` の閾値スイープ（OOF内最適化、注意: 過学習し得る）:

```bash
uv run python scripts_v2/sweep_min_p_wide_threshold_v2.py \
  --input data/oof/pl_v3_oof_w2.parquet \
  --years 2023,2024 \
  --holdout-year 2025 \
  --min-p-wide-stage candidate \
  --grid-start 0.00 --grid-stop 0.20 --grid-step 0.01 \
  --output data/backtest_v3/min_p_wide_sweep_v3_w2_2023_2024_candidate.json \
  --meta-output data/backtest_v3/min_p_wide_sweep_v3_w2_2023_2024_candidate_meta.json \
  --force
```
