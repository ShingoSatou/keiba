# スクリプトリファレンス（v3）

## 前提

- `uv sync`
- 追加依存（必要に応じて）:
  - `uv sync --extra xgboost --extra catboost`
  - `uv sync --extra torch`
- `.env` に `DATABASE_URL` を設定
- v3 の標準比較条件は `train_window_years=4` / `cv_window_policy=fixed_sliding`
- default PL contract は `meta_default`
- 実行順はこの文書を source of truth とする

## 1. v3特徴量生成

```bash
uv run python scripts_v3/build_features_v3.py \
  --input data/features_v2.parquet \
  --output data/features_v3.parquet \
  --meta-output data/features_v3_meta.json
```

補足:
- `features_v3` には final/t10 odds 列を保持する
- 既定の学習運用プロファイルは `t10_only`
- 実際の学習投入列は `feature_registry_v3.py` と contract test で制御する

補足:
- `features_v3` には final/t10 odds 列を保持する
- 学習の default operational profile は `t10_only` で、投入可否は feature registry / contract で制御する

## 2. 単勝分類（OOF + model + metrics）

```bash
uv run python scripts_v3/train_win_lgbm_v3.py --holdout-input data/features_v3_2025.parquet
uv run python scripts_v3/train_win_xgb_v3.py --holdout-input data/features_v3_2025.parquet
uv run python scripts_v3/train_win_cat_v3.py --holdout-input data/features_v3_2025.parquet
```

補足:
- binary の default feature contract は `--operational-mode t10_only`
- binary / odds calibration / PL の標準評価条件は 4年固定 sliding window
- raw entity ID（`jockey_key`, `trainer_key`）は default OFF
- 検証用に final odds を使う場合のみ `--operational-mode includes_final`
- raw entity ID を使う場合のみ `--include-entity-id-features`

主出力（例）:
- `data/oof/win_lgbm_oof.parquet`
- `data/oof/win_lgbm_cv_metrics.json`（Benter R*を含む）
- `models/win_lgbm_v3.txt`
- `models/win_lgbm_all_years_v3.txt`
- `models/win_lgbm_bundle_meta_v3.json`
- `models/win_lgbm_feature_manifest_v3.json`

## 3. 複勝分類（OOF + model + metrics）

```bash
uv run python scripts_v3/train_place_lgbm_v3.py --holdout-input data/features_v3_2025.parquet
uv run python scripts_v3/train_place_xgb_v3.py --holdout-input data/features_v3_2025.parquet
uv run python scripts_v3/train_place_cat_v3.py --holdout-input data/features_v3_2025.parquet
```

## 4. meta結合器（reference-only OOF）

```bash
uv run python scripts_v3/train_win_meta_v3.py --holdout-input data/features_v3_2025.parquet
uv run python scripts_v3/train_place_meta_v3.py --holdout-input data/features_v3_2025.parquet
```

補足:
- `p_win_meta` / `p_place_meta` は base 予測の結合器として grouped 通常CVで生成する
- group key は `race_id`
- strict temporal OOF ではない
- meta 層の内部CV指標は reference only とし、最終採否は 2025 holdout / PL holdout / wide holdout を重視する

主出力:
- `data/oof/win_meta_oof.parquet`
- `data/oof/place_meta_oof.parquet`
- `data/holdout/win_meta_holdout_pred_v3.parquet`
- `data/holdout/place_meta_holdout_pred_v3.parquet`
- `models/win_meta_v3.pkl`
- `models/place_meta_v3.pkl`

## 6. PL学習（uなし）

```bash
uv run python scripts_v3/train_pl_v3.py \
  --features-input data/features_v3.parquet \
  --holdout-input data/features_v3_2025.parquet \
  --pl-feature-profile meta_default \
  --train-window-years 4 \
  --holdout-year 2025 \
  --emit-wide-oof \
  --wide-oof-output data/oof/pl_v3_wide_oof.parquet
```

補足:
- 旧 default は raw 6本の予測列を PL に直接入力していた
- 新 default は `p_win_meta`, `p_place_meta`, `p_win_odds_t10_norm`, `PL_CONTEXT_FEATURES_SMALL`
- meta は `race_id` grouped の通常CVで作る convenience OOF
- meta OOF は strict temporal OOF ではない
- meta internal CV 指標は reference only
- binary / PL の outer evaluation policy は fixed4 を維持する
- `raw_legacy` profile で旧ルートを比較できる
- `features_v3` の numeric 列は自動収集しない
- default では calibrated odds / final odds は入れない
- final odds を学習に入れるのは `raw_legacy --include-final-odds-features` 指定時のみ（検証専用）

主出力:
- `data/oof/pl_v3_oof.parquet`
- `data/oof/pl_v3_holdout_2025_pred.parquet`
- `data/oof/pl_v3_wide_oof.parquet`（推奨のROI評価入力）
- `data/oof/pl_v3_cv_metrics.json`
- `models/pl_v3_recent_window.joblib`
- `models/pl_v3_all_years.joblib`
- `models/pl_v3_bundle_meta.json`

比較用:

```bash
uv run python scripts_v3/train_pl_v3.py \
  --features-input data/features_v3.parquet \
  --holdout-input data/features_v3_2025.parquet \
  --pl-feature-profile raw_legacy
```

## 7. 1レース推論（運用=t10）

```bash
uv run python scripts_v3/predict_race_v3.py \
  --input data/inference/race_features.parquet \
  --pl-model models/pl_v3_recent_window.joblib \
  --output data/predictions/race_v3_pred.parquet \
  --emit-wide \
  --wide-output data/predictions/race_v3_wide.parquet
```

補足:
- 入力は1レースのみ（`race_id` が1種類）を想定
- `meta_default` artifact では base model meta から `p_win_*` / `p_place_*` を生成したあと、meta artifact から `p_win_meta` / `p_place_meta` を生成して PL に渡す
- `raw_legacy` artifact では旧 raw 6本経路を維持する
- `--skip-base-model-inference` で既計算列を利用可能
- 運用パスでは final odds 依存のPLモデルを拒否（t10専用）

```bash

  --input data/oof/pl_v3_wide_oof.parquet \

  --holdout-year 2025 \
  --output data/backtest_v3/backtest_wide_v3_direct_fixed4_2020_2024.json \
  --meta-output data/backtest_v3/backtest_wide_v3_direct_fixed4_2020_2024_meta.json \
```
