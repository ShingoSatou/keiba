# スクリプトリファレンス（v3）

## 前提

- `uv sync`
- 追加依存（必要に応じて）:
  - `uv sync --extra xgboost --extra catboost`
  - `uv sync --extra torch`
- `.env` に `DATABASE_URL` を設定
- stage ごとの標準比較条件:
  - binary: `train_window_years=4`, `cv_window_policy=fixed_sliding`
  - stacker: `min_train_years=2`, `max_train_years=4`, `cv_window_policy=capped_expanding`
  - PL: `train_window_years=3`, `cv_window_policy=fixed_sliding`
- default PL contract は `stack_default`
- unsuffixed artifact / output path は `stack_default` に対応する

## 1. v3特徴量生成

```bash
uv run python scripts_v3/build_features_v3.py \
  --input data/features_v2.parquet \
  --output data/features_v3.parquet \
  --meta-output data/features_v3_meta.json
```

補足:

- `features_v3` には final, `t20`, `t15`, `t10` snapshot を保持する
- default 学習投入可否は `feature_registry_v3.py` と contract test で制御する
- default stacker では `p_win_odds_t10_norm` ではなく raw odds snapshot を使う

## 2. 単勝分類（OOF + model + metrics）

```bash
uv run python scripts_v3/train_win_lgbm_v3.py --holdout-input data/features_v3_2025.parquet
uv run python scripts_v3/train_win_xgb_v3.py --holdout-input data/features_v3_2025.parquet
uv run python scripts_v3/train_win_cat_v3.py --holdout-input data/features_v3_2025.parquet
```

補足:

- binary default feature contract は `--operational-mode t10_only`
- raw entity ID（`jockey_key`, `trainer_key`）は default OFF
- final odds は `--operational-mode includes_final` 指定時だけ使う

## 3. 複勝分類（OOF + model + metrics）

```bash
uv run python scripts_v3/train_place_lgbm_v3.py --holdout-input data/features_v3_2025.parquet
uv run python scripts_v3/train_place_xgb_v3.py --holdout-input data/features_v3_2025.parquet
uv run python scripts_v3/train_place_cat_v3.py --holdout-input data/features_v3_2025.parquet
```

## 4. strict temporal stacker

### 4.1 win stacker

```bash
uv run python scripts_v3/train_win_stack_v3.py \
  --features-input data/features_v3.parquet \
  --holdout-input data/features_v3_2025.parquet \
  --holdout-year 2025
```

### 4.2 place stacker

```bash
uv run python scripts_v3/train_place_stack_v3.py \
  --features-input data/features_v3.parquet \
  --holdout-input data/features_v3_2025.parquet \
  --holdout-year 2025
```

補足:

- `stack_win` 入力は `p_win_{lgbm,xgb,cat}` + 単勝 `t20/t15/t10` + context
- `stack_place` 入力は `p_place_{lgbm,xgb,cat}` + 複勝 `t20/t15/t10` + context
- CV は `capped_expanding(min=2, max=4)`
- 同年 fitted 値は downstream に渡さない

主出力:

- `data/oof/win_stack_oof.parquet`
- `data/oof/place_stack_oof.parquet`
- `data/holdout/win_stack_holdout_pred_v3.parquet`
- `data/holdout/place_stack_holdout_pred_v3.parquet`
- `models/win_stack_bundle_meta_v3.json`
- `models/place_stack_bundle_meta_v3.json`

## 5. grouped meta（legacy comparison）

```bash
uv run python scripts_v3/train_win_meta_v3.py --holdout-input data/features_v3_2025.parquet
uv run python scripts_v3/train_place_meta_v3.py --holdout-input data/features_v3_2025.parquet
```

補足:

- `p_win_meta` / `p_place_meta` は grouped 通常 CV の convenience OOF
- strict temporal OOF ではない
- `meta_default` profile で比較するための legacy path として残す

## 6. odds校正（任意）

```bash
uv run python scripts_v3/train_odds_calibrator_v3.py \
  --input data/features_v3.parquet \
  --score-cols p_win_odds_t10_norm,p_win_odds_final_norm \
  --methods logreg,isotonic
```

## 7. PL学習（default = stack_default）

```bash
uv run python scripts_v3/train_pl_v3.py \
  --features-input data/features_v3.parquet \
  --holdout-input data/features_v3_2025.parquet \
  --pl-feature-profile stack_default \
  --train-window-years 3 \
  --holdout-year 2025 \
  --emit-wide-oof
```

補足:

- default main path の required pred cols は `p_win_stack`, `p_place_stack`
- PL 内部では `z_win_stack`, `z_place_stack`, `place_width_log_ratio`, interaction block を使う
- `v3_pipeline_year_coverage.json` に base / stacker / PL の OOF 年対応表を出力する
- current repo の年範囲では PL fixed3 OOF fold が 0 件になるため、空 OOF + holdout/final 学習継続が正常動作

主出力:

- `data/oof/pl_v3_oof.parquet`
- `data/oof/pl_v3_holdout_2025_pred.parquet`
- `data/oof/pl_v3_wide_oof.parquet`
- `data/oof/pl_v3_cv_metrics.json`
- `data/oof/v3_pipeline_year_coverage.json`
- `models/pl_v3_recent_window.joblib`
- `models/pl_v3_all_years.joblib`
- `models/pl_v3_bundle_meta.json`

## 8. 1レース推論（運用=t10）

```bash
uv run python scripts_v3/predict_race_v3.py \
  --input data/inference/race_features.parquet \
  --pl-model models/pl_v3_recent_window.joblib \
  --output data/predictions/race_v3_pred.parquet \
  --emit-wide \
  --wide-output data/predictions/race_v3_wide.parquet
```

補足:

- 入力は 1 レースのみ
- default artifact では `base -> stacker -> PL` の順で推論する
- `stack_default` artifact は `models/win_stack_bundle_meta_v3.json`, `models/place_stack_bundle_meta_v3.json` を読んで `p_win_stack`, `p_place_stack` を作る
- `--skip-base-model-inference` で `p_win_*`, `p_place_*`, `p_win_stack`, `p_place_stack` の既計算列を使える
- final odds 依存の PL artifact は拒否する
