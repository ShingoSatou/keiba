# スクリプトリファレンス（v3）

## 前提

- `uv sync`
- 追加依存（必要に応じて）:
  - `uv sync --extra xgboost --extra catboost`
  - `uv sync --extra torch`
- `.env` に `DATABASE_URL` を設定
- stage ごとの標準比較条件:
  - binary: `train_window_years=4`, `cv_window_policy=fixed_sliding`
  - stacker: `min_train_years=2`, `max_train_years=4`, `cv_window_policy=capped_expanding`
  - PL: `train_window_years=3`, `cv_window_policy=fixed_sliding`
- default PL contract は `stack_default`
- unsuffixed artifact / output path は `stack_default` に対応する

## 1. v3特徴量生成

```bash
uv run python scripts_v3/build_features_v3.py \
  --input data/features_base.parquet \
  --output data/features_v3.parquet \
  --meta-output data/features_v3_meta.json
```

補足:

- `features_v3` には final, `t20`, `t15`, `t10` snapshot を保持する
- default 学習投入可否は `feature_registry_v3.py` と contract test で制御する
- default stacker は `p_win_odds_t20/t15/t10_norm`, `d_logit_win_*`, `place_mid_prob_t20/t15/t10`, `d_place_*` を使う

### 1.1 v3 TE feature set 生成

train:

```bash
uv run python scripts_v3/build_features_v3_te.py \
  --base-input data/features_v3.parquet \
  --te-source-input data/features_v2_te.parquet \
  --output data/features_v3_te.parquet \
  --meta-output data/features_v3_te_meta.json
```

holdout:

```bash
uv run python scripts_v3/build_features_v3_te.py \
  --base-input data/features_v3_2025.parquet \
  --te-source-input data/features_v2_te_2025.parquet \
  --output data/features_v3_te_2025.parquet \
  --meta-output data/features_v3_te_2025_meta.json
```

補足:

- safe TE extra 列の選別は `feature_registry_v3.py` に従う
- join key は `race_id`, `horse_id`, `horse_no`
- `features_v3_te*.parquet` は current `features_v3*` の全列を保持したまま、TE extra 列だけを追加する
- binary の tuned replay / Optuna `feature_set=te` はこの入口で生成した parquet を前提にする

補足:

## 3.1 binary Optuna tuning

前提:

- Optuna: `uv sync --extra optuna`
- XGBoost/CatBoost を使う場合:
  - `uv sync --extra optuna --extra xgboost`
  - `uv sync --extra optuna --extra catboost`
- `feature_set` は trial ごとに `base` / `te` を切り替える
- fixed contract:
  - `train_window_years=4`
  - `operational_mode=t10_only`
  - `include_entity_id_features=false`
- `--input-te` の再学習 replay は、basename に `_te` を含む入力パスを使う

代表コマンド:

```bash
uv run python scripts_v3/tune_binary_optuna_v3.py \
  --task win \
  --model lgbm \
  --input-base data/features_v3.parquet \
  --input-te data/features_v3_te.parquet \
  --holdout-year 2025 \
  --n-trials 200

uv run python scripts_v3/tune_binary_optuna_v3.py \
  --task win \
  --model cat \
  --input-base data/features_v3.parquet \
  --input-te data/features_v3_te.parquet \
  --holdout-year 2025 \
  --n-trials 200

uv run python scripts_v3/tune_binary_optuna_v3.py \
  --task place \
  --model xgb \
  --input-base data/features_v3.parquet \
  --input-te data/features_v3_te.parquet \
  --holdout-year 2025 \
  --n-trials 200
```

補足:

- default study 名は `binary_v3_{task}_{model}`
- default 出力:
  - `data/oof/binary_v3_{task}_{model}_trials.parquet`
  - `data/oof/binary_v3_{task}_{model}_best.json`
  - `data/oof/binary_v3_{task}_{model}_best_params.json`
  - `data/optuna/binary_v3_{task}_{model}.sqlite3`
- objective は `win` / `place` とも mean CV logloss 最小化
- `win` は baseline（default params + `feature_set=base`）に対して Benter R* 制約を満たす COMPLETE trial のうち logloss 最小を採用する
- `best_params_output` は `train_binary_model_v3.py` を同一 input / fixed contract / best params で再学習するための JSON
- 並列実行時は `V3_MODEL_THREADS=<n>` で 1 trial あたりの model thread 数を制御できる

2026-03-07 時点の運用メモ:

- TE leak 修正後の `win_xgb` 再学習 logloss は `0.2086504017556902` で、修正前 tuned best `0.2085696064918138` との差分は `+0.00008079526387638758` に留まった。
- feature 追加も検討中のため、この時点では再 tuning を見送り、探索空間の境界監査結果だけを保存する。
- 現行 6 study では、明確な上限張り付きは確認されていない。
- 下限側の圧力が強い候補:
  - `place_xgb`: `max_depth=2` が top20 全件で下限、`subsample` も top20 全件が下側 10% 以内、`colsample_bytree` / `reg_alpha` / `reg_lambda` もかなり下寄り
  - `win_xgb`: Benter 制約通過 top20 全件で `max_depth=2`、`gamma` も全件が下側 10% 以内
  - `win_cat`: Benter 制約通過 top20 の 19/20 が `depth=4`、`rsm` も 12/20 が下側 10% 以内
- 下限側の圧力が弱めだが次点で見る候補:
  - `place_lgbm`: `learning_rate` が下寄り
  - `win_lgbm`: `bagging_fraction` が下寄り
- 次回、feature set の追加後に再 tuning する場合の widening 候補:
  - `xgb`: `max_depth 1-10`, `gamma` は `0` 許容、`subsample 0.4-1.0`, `colsample_bytree 0.4-1.0`, `reg_alpha` / `reg_lambda` は `0` 許容
  - `cat`:`depth 3-10`, `rsm 0.4-1.0`
  - `lgbm`: 必要なら `learning_rate` 下限を `0.005`, `bagging_fraction` 下限を `0.5`

## 4. strict temporal stacker

### 4.1 win stacker

```bash
uv run python scripts_v3/train_win_stack_v3.py \
  --features-input data/features_v3.parquet \
  --holdout-input data/features_v3_2025.parquet \
  --holdout-year 2025
```

### 4.2 place stacker

```bash
uv run python scripts_v3/train_place_stack_v3.py \
  --features-input data/features_v3.parquet \
  --holdout-input data/features_v3_2025.parquet \
  --holdout-year 2025
```

補足:

- binary default feature contract は `BINARY_BASE_FEATURES`。market odds は default binary に入れない
- `stack_win` 入力は `p_win_{lgbm,xgb,cat}` + `p_win_odds_t20/t15/t10_norm` + `d_logit_win_*` + context
- `stack_place` 入力は `p_place_{lgbm,xgb,cat}` + `place_mid_prob_t20/t15/t10` + `place_width_log_ratio_t20/t15/t10` + `d_place_*` + context
- CV は `capped_expanding(min=2, max=4)`
- 同年 fitted 値は downstream に渡さない

主出力:

- `data/oof/win_stack_oof.parquet`
- `data/oof/place_stack_oof.parquet`
- `data/holdout/win_stack_holdout_pred_v3.parquet`
- `data/holdout/place_stack_holdout_pred_v3.parquet`
- `models/win_stack_bundle_meta_v3.json`
- `models/place_stack_bundle_meta_v3.json`

## 7. PL学習（default = stack_default）

```bash
uv run python scripts_v3/train_pl_v3.py \
  --features-input data/features_v3.parquet \
  --holdout-input data/features_v3_2025.parquet \
  --pl-feature-profile stack_default \
  --train-window-years 3 \
  --holdout-year 2025 \
  --emit-wide-oof
```

補足:

- default main path の required pred cols は `p_win_stack`, `p_place_stack`
- PL 内部では `z_win_stack`, `z_place_stack`, `place_width_log_ratio`, interaction / race-relative block を使う

主出力:
